## v1 Requirements — COMPLETE

### Models — 5/5 Complete

- [x] **MOD-01**: Developer can define a semantic view model via metaclass: `class Sales(SemanticView, view='sales')` — **Phase 1, Implemented**
- [x] **MOD-02**: Developer can declare Metric fields on a model: `revenue = Metric()` — **Phase 1, Implemented**
- [x] **MOD-03**: Developer can declare Dimension fields on a model: `country = Dimension()` — **Phase 1, Implemented**
- [x] **MOD-04**: Developer can declare Fact fields on a model: `unit_price = Fact()` — **Phase 1, Implemented**
- [x] **MOD-05**: Developer can reference fields as Python attributes for type-safe queries: `Sales.revenue` — **Phase 1, Implemented**

### Query Builder — 8/8 Complete

- [x] **QRY-01**: Developer can select metrics via `.metrics(Sales.revenue, Sales.cost)` — **Phase 2, Implemented**
- [x] **QRY-02**: Developer can select dimensions via `.dimensions(Sales.country, Sales.region)` — **Phase 2, Implemented**
- [x] **QRY-03**: Developer can select facts via `.dimensions(Sales.unit_price)` (facts behave like dimensions) — **Phase 2, Implemented**
- [x] **QRY-04**: Developer can filter results via `.filter(Q(country='US') | Q(country='CA'))` — **Phase 2, Implemented**
- [x] **QRY-05**: Developer can order results via `.order_by(Sales.revenue)` — **Phase 2, Implemented**
- [x] **QRY-06**: Developer can limit results via `.limit(100)` — **Phase 2, Implemented**
- [x] **QRY-07**: Query builder is immutable — each method returns a new query, original is unchanged — **Phase 2, Implemented**
- [x] **QRY-08**: Developer can compose Q-objects with `&` (AND), `|` (OR), `~` (NOT) operators — **Phase 2, Implemented**

### SQL Generation — 5/5 Complete

- [x] **SQL-01**: Developer can inspect generated SQL via `.to_sql()` without executing — **Phase 3, Implemented**
- [x] **SQL-02**: Snowflake SQL wraps metrics in `AGG()`: `SELECT dim, AGG(metric) FROM view` — **Phase 3, Implemented**
- [x] **SQL-03**: Databricks SQL wraps metrics in `MEASURE()`: `SELECT dim, MEASURE(metric) FROM view` — **Phase 3, Implemented**
- [x] **SQL-04**: GROUP BY is implicit — derived from selected dimensions — **Phase 3, Implemented**
- [x] **SQL-05**: SQL identifiers are properly quoted to prevent injection — **Phase 3, Implemented**

### Execution & Results — 3/3 Complete

- [x] **EXE-01**: Developer can execute query via `.fetch()` returning a list of Row objects — **Phase 4, Implemented**
- [x] **EXE-02**: Row objects support attribute access: `row.revenue` — **Phase 4, Implemented**
- [x] **EXE-03**: Row objects support dict-style access: `row['revenue']` — **Phase 4, Implemented**

### Engine Interface — 2/2 Complete

- [x] **ENG-01**: Engine ABC base class defines the interface for all backends — **Phase 3, Implemented**
- [x] **ENG-02**: MockEngine enables testing queries without a warehouse connection — **Phase 3, Implemented**

### Snowflake Backend — 1/1 Complete

- [x] **ENG-03**: SnowflakeEngine connects to Snowflake and executes queries with AGG() syntax — **Phase 5, Implemented**

### Databricks Backend — 1/1 Complete

- [x] **ENG-04**: DatabricksEngine connects to Databricks and executes queries with MEASURE() syntax — **Phase 6, Implemented**

### Registry — 3/3 Complete

- [x] **REG-01**: Developer can register engines by name: `cubano.register('default', engine)` — **Phase 4, Implemented**
- [x] **REG-02**: Engine resolution is lazy — resolved at `.fetch()` time, not at import time — **Phase 4, Implemented**
- [x] **REG-03**: Developer can select engine per-query via `.using('name')` — **Phase 4, Implemented**

### Packaging — 4/4 Complete

- [x] **PKG-01**: Library installs as `pip install cubano` with zero required dependencies — **Phase 7, Implemented**
- [x] **PKG-02**: Snowflake driver installs via `pip install cubano[snowflake]` — **Phase 7, Implemented**
- [x] **PKG-03**: Databricks driver installs via `pip install cubano[databricks]` — **Phase 7, Implemented**
- [x] **PKG-04**: Public API is accessible via `import cubano` — **Phase 7, Implemented**
