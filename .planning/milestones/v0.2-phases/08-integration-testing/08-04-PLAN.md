**Test structure:** Group by concern with @pytest.mark.warehouse and @pytest.mark.snowflake on all tests.

1. **TestFieldCombinations class:**
   - test_single_metric_execution: Query().metrics(Orders.order_total).limit(10).execute(using=snowflake_connection) → assert len(results) == 10, all rows have "order_total" field
   - test_multiple_metrics_execution: Query().metrics(Orders.order_total, Orders.order_count).limit(10).execute(using=snowflake_connection) → assert both fields present in all rows
   - test_metric_with_dimension_grouping: Query().metrics(Orders.order_total).dimensions(Orders.ordered_at).limit(50).execute(...) → assert both fields present, validate grouping behavior
   - test_dimension_only_execution: Query().dimensions(Customers.customer_name).limit(10).execute(...) → validate dimension-only queries work

2. **TestOrdering class:**
   - test_order_by_metric_descending: Query().metrics(Orders.order_total).order_by(Orders.order_total.desc()).limit(10).execute(...) → validate results are sorted descending (totals[i] >= totals[i+1])
   - test_order_by_dimension_ascending: Query().dimensions(Customers.customer_name).order_by(Customers.customer_name.asc()).limit(10).execute(...) → validate alphabetical ascending order
   - test_order_by_with_nulls: Query().dimensions(Customers.last_ordered_at).order_by(Customers.last_ordered_at.desc(nulls=NullsOrdering.LAST)).limit(20).execute(...) → validate null ordering behavior

3. **TestLimiting class:**
   - test_limit_small: Query().dimensions(Products.product_name).limit(5).execute(...) → assert len(results) == 5
   - test_limit_large: Query().metrics(Orders.order_total).limit(1000).execute(...) → assert len(results) <= 1000 (tests large result handling)
   - test_no_limit: Query().dimensions(Products.product_type).execute(...) → assert len(results) > 0 (validates unlimited queries work)

4. **TestEdgeCases class:**
   - test_empty_results: Query().metrics(Orders.order_total).filter(Q(order_total__lt=0)).execute(...) → assert results == [] (impossible filter, should return empty not error)
   - test_null_handling_in_results: Query().dimensions(Customers.last_ordered_at).limit(100).execute(...) → assert any(row["last_ordered_at"] is None for row in results) (validates nulls don't crash)
   - test_large_result_set: Query().metrics(Orders.order_count).limit(1000).execute(...) → assert len(results) == 1000 (validates pagination/fetch all)

5. **TestFiltering class:**
   - test_filter_boolean_true: Query().metrics(Orders.order_total).dimensions(Orders.is_food_order).filter(Q(is_food_order=True)).limit(50).execute(...) → assert all(row["is_food_order"] is True for row in results)
   - test_filter_comparison_greater_than: Query().dimensions(Customers.lifetime_spend).filter(Q(lifetime_spend__gt=100)).limit(50).execute(...) → assert all(row["lifetime_spend"] > 100 for row in results)

Import: from cubano import Query, Q; from cubano_jaffle_shop.jaffle_models import Orders, Customers, Products; from cubano.fields import NullsOrdering

**NOTE:** These tests assume jaffle-shop data exists in Snowflake warehouse. Tests should gracefully handle missing data (skip with pytest.skip if no data found) or document prerequisite setup in README.

Pattern: Use snowflake_connection fixture parameter (not direct engine registration). Execute queries with .execute(using=snowflake_connection). Validate result structure and SQL correctness, not exact row values (data may vary).

## Running Tests

### Quick Mock Tests (Default)

Run fast mock tests without warehouse connection:

\`\`\`bash
uv run pytest
\`\`\`

Mock tests complete in < 1 second and validate query builder logic.

### Integration Tests (Warehouse Required)

Run integration tests against real Snowflake warehouse:

\`\`\`bash
# Set credentials (one-time setup)
export SNOWFLAKE_ACCOUNT=your_account
export SNOWFLAKE_USER=your_user
export SNOWFLAKE_PASSWORD=your_password
export SNOWFLAKE_WAREHOUSE=your_warehouse
export SNOWFLAKE_DATABASE=JAFFLE_SHOP

# Run warehouse tests
uv run pytest -m warehouse -v
\`\`\`

Alternatively, create `.env` file:

\`\`\`bash
SNOWFLAKE_ACCOUNT=your_account
SNOWFLAKE_USER=your_user
SNOWFLAKE_PASSWORD=your_password
SNOWFLAKE_WAREHOUSE=your_warehouse
SNOWFLAKE_DATABASE=JAFFLE_SHOP
\`\`\`

### Parallel Execution

Run tests in parallel with pytest-xdist:

\`\`\`bash
uv run pytest -m warehouse -n auto
\`\`\`

Each worker runs in isolated schema (e.g., `cubano_test_gw0`, `cubano_test_gw1`).

### All Tests (Mock + Warehouse)

Run complete test suite:

\`\`\`bash
uv run pytest -m "mock or warehouse" -v
\`\`\`

## Test Markers

- `@pytest.mark.mock`: Fast tests using MockEngine (no credentials)
- `@pytest.mark.warehouse`: Integration tests against real warehouse (requires credentials)
- `@pytest.mark.snowflake`: Snowflake-specific tests
- `@pytest.mark.databricks`: Databricks-specific tests

## CI Behavior

GitHub Actions runs full suite (mock + warehouse) on every push using repository secrets for credentials.
