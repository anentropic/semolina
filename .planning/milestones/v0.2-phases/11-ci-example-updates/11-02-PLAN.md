Read `cubano-jaffle-shop/tests/conftest.py` first to see the current fixtures.

The current fixtures create a `MockEngine` and return it directly. Tests then call `engine.execute(query)`. The new pattern uses `cubano.register()` so tests can call `Model.query().execute()` without passing the engine explicitly.

**Strategy for mock fixtures:** Each fixture registers the engine under a unique name, yields that name so the test can use `.using('name')` if needed, then unregisters on teardown. Using a unique name per fixture avoids conflicts between concurrent tests (pytest-xdist).

However, looking at the test files, each test class uses a single engine (orders, customers, or products). The cleanest approach is to register as `'default'` within each test's scope, but that risks conflicts with xdist. A safer approach: use the engine fixture name as the registry name, and have tests call `.using('orders')` etc.

**Actually, the simplest clean pattern** is to register as `'default'` and use `autouse=False` per-test cleanup. But with `pytest-xdist`, concurrent tests will conflict on 'default'.

**Recommended approach:** Register with a unique name per fixture, and tests use `.using()` to select the engine. OR, use function-scope fixtures that register/unregister 'default' so each test gets a fresh registry state.

Since the main cubano tests already use `cubano.reset()` for cleanup, use `function` scope with `cubano.reset()` on teardown. Each fixture registers as `'default'` within the function scope.

Rewrite `conftest.py` as follows:

"""
Shared pytest fixtures for cubano-jaffle-shop integration tests.

Provides mock engine instances pre-loaded with jaffle-shop fixture data

registered with cubano so tests use Model.query().execute() directly.

import pytest
from fixtures.mock_data import customers_data, orders_data, products_data

import cubano
from cubano.engines.mock import MockEngine

    """
    Register MockEngine with orders fixture data as default engine.

    Registers the engine under 'default' so tests can call
    Orders.query().execute() without specifying an engine name.
    Engine is unregistered after each test.

    """

    cubano.register("default", engine)
    yield
    cubano.unregister("default")


@pytest.fixture

    """
    Register MockEngine with products fixture data as default engine.

    Registers the engine under 'default' so tests can call
    Products.query().execute() without specifying an engine name.
    Engine is unregistered after each test.
    """
    engine = MockEngine()
    engine.load("products", products_data)
    cubano.register("default", engine)
    yield
    cubano.unregister("default")

    Register MockEngine with all jaffle-shop fixture data as default engine.

    Provides comprehensive fixture for tests that query across multiple

    unregisters after each test.

    engine.load("customers", customers_data)
