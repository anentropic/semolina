<task type="auto">
  <name>Task 1: Split the single pytest step into two separate steps in ci.yml</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Read `.github/workflows/ci.yml` first to see the current exact content.

The current `test` job has a step like:

```yaml
- name: Run pytest (Cubano + doctests)
  run: uv run pytest tests/ src/ -m "mock or warehouse" -n auto -v
```

**The problem:** A single command with `-m "mock or warehouse"` filters out doctests (they have no markers), and even without the filter, a single command produces one merged output section — not the separate sections required by the locked decision.

**The fix:** Replace that single step with TWO sequential steps:

```yaml
- name: Run unit tests
  run: uv run pytest tests/ -m "mock or warehouse" -n auto -v

- name: Run doctests
  run: uv run pytest src/ --doctest-modules -v
```

**Why this achieves the requirement:**
- Each step produces its own GitHub Actions log section with its own pytest summary line
- Unit test step will show: `===== N passed, N skipped in X.Xs =====`
- Doctest step will show: `===== N passed, N skipped in X.Xs =====`
- If either step fails, CI fails — doctest failures are reported in their own section
- The `-n auto` flag on the unit test step enables parallel execution for unit tests
- The doctest step does NOT use `-n auto` (doctests run sequentially — they are fast)

**Step naming:** Use clear names:
- `Run unit tests` — or `Run unit tests (mock + warehouse)` for clarity
- `Run doctests` — or `Run doctests (src/)` for clarity

**Do NOT change:**
- The `cubano-jaffle-shop` test step — that step remains as-is (Plan 11-02 handles it)
- Any other steps, jobs, or configuration
- The marker filter on the unit test step — keep `-m "mock or warehouse"` to preserve existing behavior
