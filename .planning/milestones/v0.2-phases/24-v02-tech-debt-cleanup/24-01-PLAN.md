  <behavior>
    MockEngine.execute() behavior with filters applied:

    - `query._filters is None` → return all fixture rows (current behavior preserved)
    - `query._filters` is a Predicate → return only rows where `_eval_predicate(node, row)` is True

    `_eval_predicate(node, row)` logic (structural pattern matching):

    Composite:
    - `And(left, right)` → `_eval_predicate(left, row) and _eval_predicate(right, row)`
    - `Or(left, right)` → `_eval_predicate(left, row) or _eval_predicate(right, row)`
    - `Not(inner)` → `not _eval_predicate(inner, row)`

    Lookup leaf nodes (key resolution: `node.source if node.source is not None else node.field_name`):
    - `Exact` → `row.get(key) == node.value`
    - `NotEqual` → `row.get(key) != node.value`
    - `Gt` → `row.get(key) > node.value`
    - `Gte` → `row.get(key) >= node.value`
    - `Lt` → `row.get(key) < node.value`
    - `Lte` → `row.get(key) <= node.value`
    - `In` → `row.get(key) in node.value` (empty collection → False)
    - `Between` → `node.value[0] <= row.get(key) <= node.value[1]`
    - `IsNull(value=True)` → `row.get(key) is None`
    - `IsNull(value=False)` → `row.get(key) is not None`
    - `Like` → `str(row.get(key, "")).startswith` ... implement as simple contains (SQL LIKE '%x%'). Use `fnmatch` or treat `%x%` as contains, `x%` as startswith, `%x` as endswith. For simplicity use: `_sql_like(str(row.get(key,"")), node.value)` helper.
    - `ILike` → case-insensitive version of Like
    - `StartsWith` → `str(row.get(key, "")).startswith(str(node.value))`
    - `IStartsWith` → `str(row.get(key, "")).lower().startswith(str(node.value).lower())`
    - `EndsWith` → `str(row.get(key, "")).endswith(str(node.value))`
    - `IEndsWith` → `str(row.get(key, "")).lower().endswith(str(node.value).lower())`
    - `IExact` → `str(row.get(key, "")).lower() == str(node.value).lower()`
