## Overview

Create Jinja2 SQL templates for Snowflake CREATE SEMANTIC VIEW and Databricks
metric view YAML output. Implement the template rendering layer — Jinja2 environment
setup, custom filters, and the `render_view()` function that takes structured model
data and produces formatted, human-readable SQL (or YAML). Also define the `ModelData`
dataclass that both plans 09-02 and 09-03 share.

## Goal

Given a structured `ModelData` object (view name, docstring, fields with types and
docstrings), `render_view(model_data, backend, include_comments)` returns a
well-formatted SQL string for Snowflake or a well-formatted YAML string for
Databricks. Templates produce readable, properly indented output with no extra
blank lines from whitespace control. SQL comments (from docstrings) are included
when `include_comments=True` and omitted when `False`.

## Tasks

### Task 1: Define `ModelData` dataclass in `src/cubano/codegen/models.py`

This is the shared data contract between the introspection layer (09-03) and the
rendering layer (this plan). Define it here so both plans can depend on it without
circular imports.

```python
"""
Data models for Cubano codegen.

Defines the structured representation of SemanticView metadata
passed from the introspection layer to the template renderer.
"""

from __future__ import annotations

from dataclasses import dataclass, field


@dataclass(frozen=True)
class FieldData:
    """
    Structured representation of a single SemanticView field.

    Attributes:
        name: Python attribute name (e.g., 'revenue', 'country')
        field_type: One of 'metric', 'dimension', 'fact'
        docstring: Field-level docstring, empty string if none
    """

    name: str
    field_type: str  # 'metric' | 'dimension' | 'fact'
    docstring: str = ""


@dataclass(frozen=True)
class ModelData:
    """
    Structured representation of a SemanticView subclass for code generation.

    Attributes:
        class_name: Python class name (e.g., 'Sales')
        view_name: Warehouse view identifier from `view=` param (e.g., 'sales_view')
        docstring: Model-level class docstring, empty string if none
        metrics: Fields of type Metric, in definition order
        dimensions: Fields of type Dimension, in definition order
        facts: Fields of type Fact, in definition order
        source_file: Absolute path to the Python file containing this model
    """

    class_name: str
    view_name: str
    docstring: str = ""
    metrics: list[FieldData] = field(default_factory=list)
    dimensions: list[FieldData] = field(default_factory=list)
    facts: list[FieldData] = field(default_factory=list)
    source_file: str = ""
```

### Task 2: Create Snowflake template `src/cubano/codegen/templates/snowflake.sql.jinja2`

The template must produce valid `CREATE OR REPLACE SEMANTIC VIEW` DDL.

Key requirements from research:
- Clause order is strict: TABLES, RELATIONSHIPS (optional), FACTS, DIMENSIONS, METRICS
- At least one DIMENSION or METRIC must exist
- Use inline `-- comment` for docstrings (not COMMENT ON COLUMN — those are separate DDL)
- Pretty-printed: 4-space indentation, each field on its own line
- No trailing commas on last item in each section
- `include_comments` variable controls whether docstring comments are emitted

The Snowflake template does NOT know the underlying table name (Cubano models
only have a view name, not a source table). Generate the view with a placeholder
comment instructing the user to fill in `TABLES` and `RELATIONSHIPS`:

```jinja2
{#- Snowflake CREATE SEMANTIC VIEW template -#}
{#- Variables: model (ModelData), include_comments (bool) -#}
{%- if include_comments and model.docstring %}
-- {{ model.docstring | replace('\n', '\n-- ') }}
{%- endif %}
CREATE OR REPLACE SEMANTIC VIEW {{ model.view_name }}
    -- TODO: Add TABLES (...) clause pointing to your source table(s)
    -- TODO: Add RELATIONSHIPS (...) clause if joining multiple tables
{%- if model.facts %}
    FACTS (
{%-   for f in model.facts %}
{%-     if include_comments and f.docstring %}
        -- {{ f.docstring }}
{%-     endif %}
        {{ f.name }} = {{ f.name }}{{ "," if not loop.last else "" }}
{%-   endfor %}
    )
{%- endif %}
{%- if model.dimensions %}
    DIMENSIONS (
{%-   for d in model.dimensions %}
{%-     if include_comments and d.docstring %}
        -- {{ d.docstring }}
{%-     endif %}
        {{ d.name }} = {{ d.name }}{{ "," if not loop.last else "" }}
{%-   endfor %}
    )
{%- endif %}
{%- if model.metrics %}
    METRICS (
{%-   for m in model.metrics %}
{%-     if include_comments and m.docstring %}
        -- {{ m.docstring }}
{%-     endif %}
        {{ m.name }} = AGG({{ m.name }}){{ "," if not loop.last else "" }}
{%-   endfor %}
    )
{%- endif %}
;
```

For the `-- TODO` lines regarding TABLES and RELATIONSHIPS: these are essential
for the user because Cubano's `SemanticView` abstraction doesn't capture source
table names. The generated SQL is a starting point the user completes.

If a model has neither dimensions nor metrics (only facts), that is unusual but
should still render — do not error in the template.

### Task 3: Create Databricks template `src/cubano/codegen/templates/databricks.yaml.jinja2`

Databricks metric views use YAML as the primary format (per research). Generate
YAML that matches the Databricks metric view specification:

```jinja2
{#- Databricks metric view YAML template -#}
{#- Variables: model (ModelData), include_comments (bool) -#}
{%- if include_comments and model.docstring %}
# {{ model.docstring | replace('\n', '\n# ') }}
{%- endif %}
{{ model.view_name }}:
  base_table: # TODO: Replace with schema.table_name
{%- if model.dimensions or model.facts %}
  dimensions:
{%-   for d in model.dimensions %}
    - {{ d.name }}:
        expr: {{ d.name }}
{%-     if include_comments and d.docstring %}
        description: "{{ d.docstring | replace('"', '\\"') }}"
{%-     endif %}
{%-   endfor %}
{%-   for f in model.facts %}
    - {{ f.name }}:
        expr: {{ f.name }}
{%-     if include_comments and f.docstring %}
        description: "{{ f.docstring | replace('"', '\\"') }}"
{%-     endif %}
{%-   endfor %}
{%- endif %}
{%- if model.metrics %}
  measures:
{%-   for m in model.metrics %}
    - {{ m.name }}:
        expr: SUM({{ m.name }})  # TODO: Replace SUM with correct aggregation
{%-     if include_comments and m.docstring %}
        description: "{{ m.docstring | replace('"', '\\"') }}"
{%-     endif %}
{%-   endfor %}
{%- endif %}
```

Note the `SUM({{ m.name }})` placeholder: Cubano `Metric()` fields don't capture
aggregation type, so generate `SUM()` with a comment asking the user to fill in
the correct aggregation. This is honest and safe.

### Task 4: Create `src/cubano/codegen/renderer.py` — Jinja2 rendering layer

```python
"""
Jinja2-based SQL/YAML renderer for Cubano codegen.

Loads templates from the bundled templates/ directory, applies custom
filters, and renders ModelData objects to formatted SQL or YAML strings.
"""

from __future__ import annotations

from pathlib import Path

from jinja2 import Environment, FileSystemLoader, StrictUndefined

from cubano.codegen.models import ModelData

_TEMPLATE_DIR = Path(__file__).parent / "templates"

# Map backend names to template filenames
_BACKEND_TEMPLATES: dict[str, str] = {
    "snowflake": "snowflake.sql.jinja2",
    "databricks": "databricks.yaml.jinja2",
}


def _make_environment() -> Environment:
    """
    Create a Jinja2 Environment with strict undefined and SQL-safe settings.

    Returns:
        Configured Jinja2 Environment loading from bundled templates directory.
    """
    return Environment(
        loader=FileSystemLoader(str(_TEMPLATE_DIR)),
        undefined=StrictUndefined,
        # Keep_trailing_newline ensures output ends with a newline
        keep_trailing_newline=True,
        # Do NOT use autoescape — SQL/YAML is not HTML
        autoescape=False,
    )


def render_view(
    model: ModelData,
    backend: str,
    include_comments: bool = True,
) -> str:
    """
    Render a single SemanticView model to SQL or YAML using the appropriate template.

    Args:
        model: Structured model metadata extracted from the Python class.
        backend: Target backend name. Must be one of 'snowflake', 'databricks'.
        include_comments: Whether to emit docstring comments in the output.

    Returns:
        Rendered SQL (Snowflake) or YAML (Databricks) string.

    Raises:
        ValueError: If backend is not recognised.
        jinja2.TemplateNotFound: If the template file is missing (internal error).
        jinja2.UndefinedError: If a template variable is missing (internal error).
    """
    if backend not in _BACKEND_TEMPLATES:
        supported = ", ".join(sorted(_BACKEND_TEMPLATES))
        raise ValueError(f"Unknown backend {backend!r}. Supported: {supported}")

    env = _make_environment()
    template_name = _BACKEND_TEMPLATES[backend]
    template = env.get_template(template_name)

    return template.render(
        model=model,
        include_comments=include_comments,
    )


def render_file_header(source_file: str, backend: str) -> str:
    """
    Return a file-level header comment to prepend to generated output.

    Helps users identify where the SQL came from.

    Args:
        source_file: Absolute path to the source Python file.
        backend: Backend name ('snowflake' or 'databricks').

    Returns:
        Comment string suitable for prepending to output.
    """
    comment_char = "--" if backend == "snowflake" else "#"
    return (
        f"{comment_char} Generated by cubano codegen\n"
        f"{comment_char} Source: {source_file}\n"
        f"{comment_char} Backend: {backend}\n"
    )
```

### Task 5: Write unit tests for the renderer

Create `tests/codegen/test_renderer.py`:

```python
"""Unit tests for the Jinja2 template renderer."""

import pytest
from cubano.codegen.models import FieldData, ModelData
from cubano.codegen.renderer import render_view, render_file_header


@pytest.fixture
def sales_model() -> ModelData:
    """A simple SemanticView model for rendering tests."""
    return ModelData(
        class_name="Sales",
        view_name="sales_view",
        docstring="Sales semantic view with revenue and country.",
        metrics=[
            FieldData(name="revenue", field_type="metric", docstring="Total revenue."),
            FieldData(name="cost", field_type="metric", docstring=""),
        ],
        dimensions=[
            FieldData(name="country", field_type="dimension", docstring="Customer country."),
            FieldData(name="region", field_type="dimension", docstring=""),
        ],
        facts=[
            FieldData(name="unit_price", field_type="fact", docstring="Price per unit."),
        ],
        source_file="/tmp/models.py",
    )


class TestSnowflakeRenderer:
    def test_contains_create_semantic_view(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake")
        assert "CREATE OR REPLACE SEMANTIC VIEW sales_view" in sql

    def test_dimensions_section_present(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake")
        assert "DIMENSIONS" in sql
        assert "country = country" in sql
        assert "region = region" in sql

    def test_metrics_section_present(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake")
        assert "METRICS" in sql
        assert "revenue = AGG(revenue)" in sql
        assert "cost = AGG(cost)" in sql

    def test_facts_section_present(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake")
        assert "FACTS" in sql
        assert "unit_price = unit_price" in sql

    def test_docstring_comments_included_by_default(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake", include_comments=True)
        assert "-- Sales semantic view" in sql
        assert "-- Total revenue." in sql
        assert "-- Customer country." in sql

    def test_no_comments_flag_strips_comments(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake", include_comments=False)
        assert "-- Sales semantic view" not in sql
        assert "-- Total revenue." not in sql

    def test_no_trailing_comma_on_last_field(self, sales_model: ModelData) -> None:
        sql = render_view(sales_model, "snowflake")
        # The last metric should not have a trailing comma
        lines = sql.splitlines()
        metric_lines = [l for l in lines if "AGG(" in l]
        assert metric_lines[-1].rstrip().endswith(")")

    def test_model_without_docstring(self) -> None:
        model = ModelData(
            class_name="Bare",
            view_name="bare_view",
            metrics=[FieldData(name="count", field_type="metric")],
            dimensions=[FieldData(name="country", field_type="dimension")],
        )
        sql = render_view(model, "snowflake")
        assert "CREATE OR REPLACE SEMANTIC VIEW bare_view" in sql


class TestDatabricksRenderer:
    def test_contains_view_name_key(self, sales_model: ModelData) -> None:
        yaml_out = render_view(sales_model, "databricks")
        assert "sales_view:" in yaml_out

    def test_dimensions_section_present(self, sales_model: ModelData) -> None:
        yaml_out = render_view(sales_model, "databricks")
        assert "dimensions:" in yaml_out
        assert "- country:" in yaml_out

    def test_measures_section_present(self, sales_model: ModelData) -> None:
        yaml_out = render_view(sales_model, "databricks")
        assert "measures:" in yaml_out
        assert "- revenue:" in yaml_out
        assert "expr: SUM(revenue)" in yaml_out

    def test_docstring_comments_included(self, sales_model: ModelData) -> None:
        yaml_out = render_view(sales_model, "databricks", include_comments=True)
        assert 'description: "Total revenue."' in yaml_out

    def test_no_comments_strips_descriptions(self, sales_model: ModelData) -> None:
        yaml_out = render_view(sales_model, "databricks", include_comments=False)
        assert 'description:' not in yaml_out


class TestRendererEdgeCases:
    def test_unknown_backend_raises_value_error(self) -> None:
        model = ModelData(class_name="X", view_name="v")
        with pytest.raises(ValueError, match="Unknown backend"):
            render_view(model, "mysql")

    def test_render_file_header_snowflake(self) -> None:
        header = render_file_header("/tmp/models.py", "snowflake")
        assert header.startswith("--")
        assert "snowflake" in header

    def test_render_file_header_databricks(self) -> None:
        header = render_file_header("/tmp/models.py", "databricks")
        assert header.startswith("#")
        assert "databricks" in header

    def test_model_with_only_metrics(self) -> None:
        model = ModelData(
            class_name="MetricOnly",
            view_name="metric_view",
            metrics=[FieldData(name="revenue", field_type="metric")],
        )
        sql = render_view(model, "snowflake")
        assert "METRICS" in sql
        assert "DIMENSIONS" not in sql
        assert "FACTS" not in sql
```
