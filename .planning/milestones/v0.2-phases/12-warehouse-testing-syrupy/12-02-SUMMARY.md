## Decisions Made

- **Type annotations on test functions:** basedpyright strict mode requires parameter type annotations even on test functions. Used `Any` for `backend_engine` (complex union type from fixture) and `SnapshotAssertion` for `snapshot` (under TYPE_CHECKING).
- **noqa: ARG001 for backend_engine:** The parameter is accepted for its side effect (engine registration) but not called in the test body. Ruff ARG001 would flag it as unused without the noqa comment.
- **MockEngine bootstrap for snapshot generation:** Without warehouse credentials, snapshots were generated by temporarily forcing `is_recording = False` in conftest fixtures, running `--snapshot-update`, then restoring conftest via `git checkout`. This produces a valid baseline for CI replay.
