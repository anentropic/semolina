  <files>src/cubano/engines/sql.py, src/cubano/engines/snowflake.py, src/cubano/engines/databricks.py, src/cubano/engines/mock.py, tests/unit/test_sql.py</files>
  <behavior>
    Compiler input/output contract:
    - _compile_predicate(node: Predicate) -> tuple[str, list[Any]]
    - Returns (sql_fragment, params_list)

    Leaf node compilation (each using dialect.placeholder):
    - Exact(f, v) -> ("{quote(f)} = {ph}", [v])
    - Gt(f, v) -> ("{quote(f)} > {ph}", [v])
    - Gte(f, v) -> ("{quote(f)} >= {ph}", [v])
    - Lt(f, v) -> ("{quote(f)} < {ph}", [v])
    - Lte(f, v) -> ("{quote(f)} <= {ph}", [v])
    - In(f, []) -> ("1 = 0", [])  # empty IN = always false
    - In(f, [a,b,c]) -> ("{quote(f)} IN ({ph}, {ph}, {ph})", [a, b, c])
    - Between(f, (lo,hi)) -> ("{quote(f)} BETWEEN {ph} AND {ph}", [lo, hi])
    - IsNull(f, True) -> ("{quote(f)} IS NULL", [])
    - IsNull(f, False) -> ("{quote(f)} IS NOT NULL", [])
    - Like(f, v) -> ("{quote(f)} LIKE {ph}", [v])
    - ILike(f, v) -> ("{quote(f)} ILIKE {ph}", [v])
    - StartsWith(f, v) -> ("{quote(f)} LIKE {ph}", [v + "%"])  # wildcard appended to param value
    - IStartsWith(f, v) -> ("{quote(f)} ILIKE {ph}", [v + "%"])
    - EndsWith(f, v) -> ("{quote(f)} LIKE {ph}", ["%" + v])
    - IEndsWith(f, v) -> ("{quote(f)} ILIKE {ph}", ["%" + v])
    - IExact(f, v) -> ("{quote(f)} ILIKE {ph}", [v])  # no wildcards
    - Lookup() (catch-all) -> NotImplementedError

    Note on StartsWith/EndsWith: For v0.2, do NOT escape % and _ in the user value
    before appending the wildcard. This matches simpler behavior. The LIKE escaping
    question is flagged as open in RESEARCH.md and deferred to v0.3. Document this
    in a code comment.

    Composite node compilation:
    - And(l, r) -> ("({l_sql} AND {r_sql})", l_params + r_params)
    - Or(l, r) -> ("({l_sql} OR {r_sql})", l_params + r_params)
    - Not(i) -> ("NOT ({i_sql})", i_params)

    Non-predicate input:
    - case _: -> TypeError

    Dialect.placeholder property:
    - SnowflakeDialect.placeholder = "%s"
    - DatabricksDialect.placeholder = "?"
    - MockDialect.placeholder = "%s"

    SQLBuilder new methods:
    - build_select_with_params(query) -> tuple[str, list[Any]]
      Same as build_select() but returns (sql_template, params) for execution
    - _build_where_clause() rewritten: uses _compile_predicate(), returns "WHERE {sql}"
    - _build_where_clause_with_params(query) -> tuple[str, list[Any]]
      Returns ("WHERE {sql}", params) for parameterized pipeline

    Engine changes:
    - SnowflakeEngine.execute(): use build_select_with_params(), pass (sql, params) to cursor.execute(sql, params)
    - DatabricksEngine.execute(): use build_select_with_params(), pass (sql, params) to cursor.execute(sql, parameters=params)
    - MockEngine.to_sql(): use build_select() (inline rendered, same as before — SQLBuilder.build_select renders inline via render_inline helper)

    Inline rendering for to_sql() (display/debugging):
    - SQLBuilder.render_inline(sql_template, params) -> str
      Replaces each placeholder with repr(param), one at a time
    - build_select() calls build_select_with_params() then render_inline() for backward compat
    - to_sql() return type stays str (no breaking change)
  </behavior>
