<feature>
  <name>Predicate Tree IR</name>
  <files>src/cubano/filters.py, tests/unit/test_filters.py</files>
  <behavior>
    Predicate tree nodes serve as the filter intermediate representation:

    Construction:
    - Exact("country", "US") -> Exact(field_name="country", value="US")
    - Gt("revenue", 1000) -> Gt(field_name="revenue", value=1000)
    - Between("date", (start, end)) -> Between(field_name="date", value=(start, end))
    - In("country", ["US", "CA"]) -> In(field_name="country", value=["US", "CA"])
    - IsNull("region", True) -> IsNull(field_name="region", value=True)

    Composition via operators:
    - Exact(...) & Gt(...) -> And(left=Exact(...), right=Gt(...))
    - Exact(...) | Exact(...) -> Or(left=Exact(...), right=Exact(...))
    - ~Exact(...) -> Not(inner=Exact(...))
    - ~~Exact(...) -> Not(inner=Not(inner=Exact(...)))

    Immutability:
    - All dataclass nodes are frozen=True
    - Composition creates new nodes, never mutates

    Edge cases:
    - Nested NOT: ~~predicate produces Not(Not(predicate))
    - Empty In: In("field", []) is valid construction (compiler handles semantics)
    - Predicate base is NOT a dataclass (plain class with __and__, __or__, __invert__)
    - Lookup inherits from both Predicate and Generic[T]

    Type hierarchy:
    - Predicate (plain class, not dataclass)
      - And(Predicate) frozen dataclass: left, right
      - Or(Predicate) frozen dataclass: left, right
      - Not(Predicate) frozen dataclass: inner
      - Lookup(Predicate, Generic[T]) frozen dataclass: field_name, value
        - Exact(Lookup[Any])
        - Gt(Lookup[Any])
        - Gte(Lookup[Any])
        - Lt(Lookup[Any])
        - Lte(Lookup[Any])
        - In(Lookup[Any])
        - Between(Lookup[Any])
        - IsNull(Lookup[bool])
        - Like(Lookup[str])
        - ILike(Lookup[str])
        - StartsWith(Lookup[str])
        - IStartsWith(Lookup[str])
        - EndsWith(Lookup[str])
        - IEndsWith(Lookup[str])
        - IExact(Lookup[str])
  </behavior>
