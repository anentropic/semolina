<decisions_made>

- `FACTS` clause does NOT exist in Snowflake `CREATE SEMANTIC VIEW` DDL — columns go in
  DIMENSIONS. Snowflake may still return `kind=FACT` on introspection for numeric columns.
- `RELATIONSHIPS` syntax in Snowflake uses parentheses: `o (location_id) REFERENCES l (location_id)`,
  NOT dot notation `o.location_id REFERENCES l.location_id`.
- Docs corrected to reflect both of the above.
- SecretStr must be unwrapped via `.get_secret_value()` before passing to connectors.
</decisions_made>
