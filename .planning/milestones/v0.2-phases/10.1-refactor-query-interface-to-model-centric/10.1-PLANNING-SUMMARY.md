## Overview

Phase 10.1 is an urgent insertion after Phase 10 (Documentation) to refactor Cubano's query API from procedural (`Query()`) to model-centric (`Model.query()`). The phase redesigns the user-facing API for better ergonomics, IDE autocomplete, type safety, and discoverability.

**Key Decision:** No backward compatibility required — the library hasn't been released yet, so we optimize for "getting it right" over migration paths.

## Backward Compatibility

⚠️ **Breaking Changes (Intentional):**
- Query() constructor is now internal (use Model.query() instead)
- .filter() method is now internal (use .where() instead)
- .fetch() method is now internal (use .execute() instead)

✅ **Backward Compatible:**
- Old API methods still exist (for internal use and tests)
- No external API was released yet, so no external breakage
- Tests can continue using internal methods if needed

## Risk Assessment

**LOW RISK:**
- Codebase is mature (shipped in v0.1)
- All infrastructure exists (descriptors, Q objects, Row class, dataclass Query)
- Changes are primarily API-level, not architectural
- Comprehensive test coverage required before sign-off

**MEDIUM RISK:**
- Reserved method names exclude some field names (mitigated by helpful error messages)
- Namespace collision handling uses "reserved names" pattern (validated against Django ORM, Looker, Metabase precedent)

**MITIGATION:**
- Conservative reserved names list (only essential builder methods)
- Clear error messages suggesting alternatives (rename field or use source mapping)
- Documentation explaining the constraint
