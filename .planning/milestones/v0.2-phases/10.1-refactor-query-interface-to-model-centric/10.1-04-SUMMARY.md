## Decisions Made

1. **Result as lightweight wrapper:** Not a list subclass (explicit is better than implicit), proxies behavior to wrapped rows
2. **Dual execution API:** Both .fetch() (internal) and .execute() (public) support backward compatibility while providing clear user-facing API
3. **Eager execution approach:** execute() returns immediately after calling fetch(), no lazy evaluation or generators
4. **Extension-ready Result:** Designed with future helper methods in mind (no breaking changes when adding .to_pandas() etc.)
5. **Iteration/indexing proxying:** __iter__ and __getitem__ proxy to self.rows directly
