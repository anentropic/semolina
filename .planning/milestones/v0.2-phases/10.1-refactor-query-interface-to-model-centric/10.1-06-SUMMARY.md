## Issues Encountered & Resolutions

**Issue 1: Typecheck errors in Q.children access**
- Root cause: Q.children typed as `list[tuple[str, Any] | Q]`, making tuple indexing unsafe
- Solution: Added Q.is_leaf() method to check if children are tuples before accessing
- Resolution: Added isinstance checks in all test files before accessing Q.children[0][0]
- Status: ✅ Fixed, all tests pass

**Issue 2: Doctest failures in docstrings**
- Root cause: Example docstrings referenced undefined model (Users instead of Sales)
- Solution: Updated Query.execute() and Result docstrings to use Sales model from conftest
- Resolution: Updated expected result counts to match mock data (3 rows)
- Status: ✅ Fixed, all doctests pass

**Issue 3: Type: ignore needed for test classes in pytest.raises blocks**
- Root cause: Classes defined inside with pytest.raises() seen as unused by type checker
- Solution: Added type: ignore[reportUnusedClass] directives to suppress checker warnings
- Note: This is appropriate for test code where the class is intentionally defined to trigger an error
- Status: ✅ Appropriate use of type: ignore in test context
