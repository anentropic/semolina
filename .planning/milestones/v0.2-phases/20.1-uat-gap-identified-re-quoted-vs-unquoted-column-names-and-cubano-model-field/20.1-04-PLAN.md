  <name>Task 2: Format --help exit codes as Rich paragraph list</name>
  <files>

  <action>
    Update the `epilog` string in `app.command("codegen", epilog=...)` in
    `src/cubano/cli/__init__.py`.

    **Root cause:** Typer's epilog renderer collapses single newlines into spaces
    within each paragraph (see Typer source: `x.replace("\n", " ")`). Double newlines
    (`\n\n`) create paragraph breaks, which ARE preserved as separate lines. Use
    `\n\n` as the separator between each exit code row.

    Replace the current plain-text epilog:
    ```python
    epilog=(
        "Exit codes:\n"
        "  0  Success\n"
        "  1  Unexpected error\n"
        ...
    )
    ```

    With a Rich-markup version using `\n\n` between rows:
    ```python
    epilog=(
        "[bold]Exit codes[/bold]\n\n"
        "  [green]0[/green]  Success\n\n"
        "  [yellow]1[/yellow]  Unexpected error\n\n"
        "  [yellow]2[/yellow]  Invalid [bold]--backend[/bold] value (or omitted)\n\n"
        "  [red]3[/red]  View not found in the warehouse\n\n"
        "  [red]4[/red]  Connection or authentication failure"
    )
    ```

    Color convention: green=success, yellow=user-error (fixable), red=system-error
    (not user's fault). This matches common CLI conventions (e.g., git, docker).

    The `app` is created with `rich_markup_mode='rich'` by default (Rich is installed)
    — no change to the `typer.Typer(...)` constructor needed.

    No test changes required — no existing tests check the epilog string content.
  </action>
