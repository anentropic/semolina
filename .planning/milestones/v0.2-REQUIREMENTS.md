# Requirements: Cubano v0.2

**Defined:** 2026-02-17
**Core Value:** A single, Pythonic query API that works identically across Snowflake and Databricks semantic views, with typed models, IDE autocomplete, and backend-agnostic code.

## v0.2 Requirements

Requirements for Cubano v0.2: Developer tooling, integration testing, and comprehensive documentation.

### Integration Testing

- [x] **INT-01**: User can run pytest suite against real Snowflake jaffle-shop data
- [x] **INT-02**: Integration tests validate queries with various field combinations (metrics, dimensions, filters)
- [x] **INT-03**: Integration tests verify correct result ordering and limiting behavior
- [x] **INT-04**: Integration tests handle edge cases (empty results, null values, large result sets)
- [x] **INT-05**: Test suite isolates data in separate schema to prevent flakiness
- [x] **INT-06**: Warehouse credentials loaded from environment (not hardcoded)

### Codegen

<!-- SUPERSEDED: CODEGEN-01–08 describe forward codegen (Python → SQL/YAML) that was
intentionally deleted in Phase 20-04 (reverse codegen teardown). The currently shipped
behavior is reverse codegen (warehouse → Python), tracked as CODEGEN-REVERSE and
CODEGEN-WAREHOUSE in the Traceability table below. -->

- [x] **CODEGEN-01**: User can run `cubano codegen` CLI to generate SQL from Cubano model files
- [x] **CODEGEN-02**: Codegen generates CREATE SEMANTIC VIEW SQL (Snowflake syntax with AGG wrapping)
- [x] **CODEGEN-03**: Codegen generates metric view definitions for Databricks (MEASURE syntax)
- [x] **CODEGEN-04**: Codegen processes all models in a file, outputting multiple CREATE statements
- [x] **CODEGEN-05**: Generated code passes syntax validation (`py_compile`, import checks) before writing
- [x] **CODEGEN-06**: Codegen validates no circular relationships in generated SQL
- [x] **CODEGEN-07**: User specifies output file and target backend (`--output path`, `--backend snowflake|databricks|both`)
- [x] **CODEGEN-08**: Generated SQL is formatted for readability and includes field comments from model docstrings

### Documentation

- [x] **DOCS-01**: Getting started guide with install, model definition, first query example
- [x] **DOCS-02**: API reference auto-generated from docstrings (MkDocs + mkdocstrings)
- [x] **DOCS-03**: Query language guide: `.metrics()`, `.dimensions()`, `.where()`, `.order_by()`, `.limit()`
- [x] **DOCS-04**: Query language guide: Predicate composition with `&`, `|`, `~` operators
- [x] **DOCS-05**: Backend comparison: Snowflake vs Databricks differences and migration tips
- [x] **DOCS-06**: MkDocs + Material theme configured locally
- [x] **DOCS-07**: Explore and integrate useful mkdocs plugins (search, awesome-pages, macros, etc.)
- [x] **DOCS-08**: GitHub Actions workflow builds docs on push to main
- [x] **DOCS-09**: Built docs auto-deploy to GitHub Pages
- [x] **DOCS-10**: Examples in docstrings validated with doctest (fail on broken examples)

### Warehouse Integration Testing (brought forward from v1+)

- [x] **TEST-VCR**: Record/replay warehouse queries with snapshot-based testing to enable real warehouse tests in CI without cost

## v1+ Requirements (Deferred)

Acknowledged but not in v0.2 scope:

### Advanced Query Features

- **QRY-WINDOW**: Window functions (ROW_NUMBER, LAG, LEAD, RANK)
- **QRY-CTE**: Common table expressions (WITH clauses)
- **QRY-JOIN**: Multi-view joins and subqueries
- **QRY-UNION**: UNION / UNION ALL for combining result sets
- **QRY-HAVING**: HAVING clause for metric filtering

### Output & Analysis

- **OUT-DATAFRAME**: `fetch_df()` returning pandas DataFrame
- **OUT-PLOT**: Result visualization helpers (matplotlib/plotly integration)
- **OUT-EXPORT**: Export to CSV, Parquet, JSON formats
- **ASYNC-FETCH**: Async query execution (`async def fetch()`)

### Codegen Enhancements

- **CODEGEN-DBT**: Generate Cubano models from dbt manifests
- **CODEGEN-WAREHOUSE**: Generate models from warehouse introspection (live schema scanning)
- **CODEGEN-REVERSE**: Generate Cubano Python model class from warehouse semantic view introspection (warehouse→Python direction; implemented in Phase 20 as `cubano codegen <schema.view_name>`)

### Integration Test Enhancements

- **TEST-VCR**: Record/replay warehouse queries to reduce CI costs
- **TEST-PERF**: Performance regression testing and baselines
- **TEST-MULTI**: Multi-warehouse testing (Snowflake + Databricks in same run)

### Documentation Enhancements

- **DOCS-VERSIONING**: Versioned documentation (v0.1, v0.2 separate sites)
- **DOCS-TUTORIAL**: Interactive tutorial / Jupyter notebooks
- **DOCS-VIDEO**: Video walkthroughs of key features

## Out of Scope

Explicitly excluded from v0.2:

| Feature | Reason |
|---------|--------|
| dbt manifest → Cubano model codegen | Deferred to v0.3; focus v0.2 on Cubano → SQL direction |
| Django integration (`cubano-django`) | Separate package, later milestone |
| DataFrame returns (`fetch_df()`) | Output format enhancement, defer to v0.3 |
| Async query support | Architecture change, evaluate after v0.2 stability |
| Multi-view joins | Complex feature, requires extensive design work |
| Window functions | SQL complexity increases, table-stakes only |
| Reverse codegen (models → YAML) | Lower priority, explore after v0.2 |
| VCR-based test recording | Cost optimization, add if CI costs become concern |
