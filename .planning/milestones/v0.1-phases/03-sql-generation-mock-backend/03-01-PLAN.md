Create new file src/cubano/engines/sql.py with Dialect ABC and concrete implementations.

Structure:
1. Dialect ABC with two abstract methods:
   - quote_identifier(name: str) -> str: Quote a column/table name
   - wrap_metric(field_name: str) -> str: Wrap a metric field (AGG vs MEASURE)

2. SnowflakeDialect(Dialect):
   - quote_identifier: Use double quotes ("), escape internal " as ""
   - wrap_metric: Return f"AGG({self.quote_identifier(field_name)})"

3. DatabricksDialect(Dialect):
   - quote_identifier: Use backticks (`), escape internal ` as ``
   - wrap_metric: Return f"MEASURE({self.quote_identifier(field_name)})"

4. MockDialect(Dialect):
   - quote_identifier: Use double quotes like Snowflake (")
   - wrap_metric: Use AGG() like Snowflake (MockEngine uses Snowflake-like syntax for test data)

Each dialect class should:
- Have a class docstring explaining when to use it
- Include method docstrings with examples
- Show escaping examples in docstrings (e.g., "my\"field" -> "my""field")

Add comprehensive docstrings to Dialect ABC explaining:
- quote_identifier() is for identifiers (table/column names), not string literals
- wrap_metric() is dialect-specific (AGG vs MEASURE)
- Both methods must be called by SQL generator

Test escaping logic by example in docstrings:
- Snowflake: Column "my"name" (with internal quote) -> "my""name" (double the quote)
- Databricks: Column `my`name` -> `my``name` (same pattern with backticks)
