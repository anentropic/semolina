Create new file src/cubano/engines/mock.py with MockEngine class.

MockEngine(Engine) provides:

1. __init__(self, fixtures: dict[str, list[dict]] | None = None):
   - fixtures: Optional dict mapping view_name -> list of row dicts
   - Example: {'sales_view': [{'revenue': 1000, 'country': 'US'}, ...]}
   - Default to empty dict if None
   - Store self.fixtures = fixtures or {}
   - Create and store self.dialect = MockDialect()

2. to_sql(self, query: Query) -> str:
   - Call query._validate_for_execution() to ensure query is valid
   - Create SQLBuilder(self.dialect)
   - Call builder.build_select(query)
   - Return SQL string
   - This is identical to Query.to_sql() but uses engine's dialect

3. execute(self, query: Query) -> list[dict]:
   - Call query._validate_for_execution()
   - Extract view name from query (first field's owner model)
   - Look up fixtures[view_name] (or empty list if not found)
   - Return the fixture data as-is (no filtering/aggregation in Phase 3)
   - For now: Return list of dicts. Phase 4 will add Row objects.

Implementation notes:
- Extract view name: query._metrics[0].owner._view_name or query._dimensions[0].owner._view_name
  (guaranteed to exist after _validate_for_execution())
- Handle empty fixtures gracefully: Return [] if view not in fixtures
- Docstrings for class and methods
- Type hints using forward references (from __future__ import annotations)

Phase 3 note: MockEngine returns raw fixture data without filtering or aggregation.
The Phase 5/6 tests will validate filtering/aggregation against real backends.
For Phase 3, MockEngine validates structure and returns test data.
