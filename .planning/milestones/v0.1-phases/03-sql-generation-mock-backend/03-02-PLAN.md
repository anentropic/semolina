  <action>
Add SQLBuilder class to the end of src/cubano/engines/sql.py (after existing Dialect classes).

SQLBuilder takes a Dialect instance in __init__ and provides a build_select(query: Query) -> str method.

Structure (method chaining pattern, not actual methods on builder, but logical sections):

1. __init__(self, dialect: Dialect):
   Store dialect as instance variable.

2. build_select(self, query: Query) -> str:
   Main public method that orchestrates SQL generation.
   Build SQL as list of strings, join with "\n" at end:
   - SELECT clause (via _build_select_clause)
   - FROM clause (via _build_from_clause)
   - WHERE clause (optional, via _build_where_clause if query._filters)
   - GROUP BY clause (optional, via _build_group_by_clause if query._dimensions)
   - ORDER BY clause (optional, via _build_order_by_clause if query._order_by_fields)
   - LIMIT clause (optional, via _build_limit_clause if query._limit_value)

3. _build_select_clause(self, query: Query) -> str:
   Return "SELECT " + comma-separated list of:
   - For Metrics: dialect.wrap_metric(metric.name)
   - For Dimensions/Facts: dialect.quote_identifier(dim.name)
   Example: "SELECT AGG(\"revenue\"), \"country\""

4. _build_from_clause(self, query: Query) -> str:
   Extract view name from query._metrics[0].owner._view_name (first field's owner model)
   Return "FROM " + dialect.quote_identifier(view_name)
   Example: "FROM \"sales_view\""
   Handle case where query has only dimensions (no metrics): Use first dimension's owner model

5. _build_where_clause(self, query: Query) -> str:
   Convert query._filters (Q object) to WHERE clause.
   For now: Return "WHERE " + repr(query._filters) or similar placeholder.
   NOTE: Q-object rendering to SQL is complex and deferred to Phase 4 (where filter execution happens).
   For Phase 3 SQL generation, just show structure - e.g., "WHERE <filters>" or convert to simple representation.
   Store the filter representation but don't try to fully evaluate it.

6. _build_group_by_clause(self, query: Query) -> str:
   If query._dimensions exists: Return "GROUP BY ALL"
   (Both Snowflake and Databricks support this; automatically groups by non-aggregated SELECT columns)

7. _build_order_by_clause(self, query: Query) -> str:
   Build "ORDER BY " + comma-separated list:
   - For Field (bare): dialect.quote_identifier(field.name) + " ASC"
   - For OrderTerm: dialect.quote_identifier(field.name) + (" DESC" if descending else " ASC")
     + (" NULLS FIRST" if nulls == NullsOrdering.FIRST else " NULLS LAST" if nulls == NullsOrdering.LAST else "")
   Example: "ORDER BY \"revenue\" DESC NULLS FIRST, \"country\" ASC"

8. _build_limit_clause(self, query: Query) -> str:
   Return "LIMIT " + str(query._limit_value)
   Example: "LIMIT 100"

Important implementation notes:
- Metrics come from query._metrics, Dimensions/Facts from query._dimensions
- Each field has .name attribute (set during Field.__set_name__)
- Each field has .owner attribute pointing to the SemanticView model class
- SemanticView classes have ._view_name attribute (set in __init_subclass__)
- OrderTerm has .field, .descending, .nulls attributes
- For WHERE clause: Store Q object representation as placeholder (full SQL generation deferred to Phase 4)
  For now, you can use: "WHERE 1=1" or "WHERE <query._filters>" just to show structure.
  Actual Q-to-SQL translation requires a separate query filter compiler (Phase 4).

- All methods return str
- Signature: build_select(self, query: Query) -> str

Replace Query.to_sql() stub with real implementation using SQLBuilder.

Current stub:
```python
def to_sql(self) -> str:
    self._validate_for_execution()
    raise NotImplementedError("SQL generation in Phase 3")
```

New implementation:
```python
def to_sql(self) -> str:
    self._validate_for_execution()
    from .engines.sql import SQLBuilder, MockDialect
    builder = SQLBuilder(MockDialect())
    return builder.build_select(self)
```

Rationale for MockDialect:
- Query.to_sql() is for inspection/debugging (no backend specified yet)
- Use MockDialect (Snowflake-like double quotes, AGG() wrapping) as sensible default
- Phase 4 will add engine registry and .using() method for backend selection
- Engine.to_sql() (03-02 task adds) will use the engine's own dialect for production

Docstring update:
- Remove mention of "Phase 3" and NotImplementedError
- Update docstring to explain: "Generates SQL using MockDialect (Snowflake-like) for inspection. Use Engine.to_sql(query) for backend-specific SQL."
