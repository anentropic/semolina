  <name>Create release workflow with build, validate, publish jobs</name>
  <files>.github/workflows/release.yml</files>
  <action>
Create GitHub Actions workflow `.github/workflows/release.yml` with the following configuration:

**Triggers:**
- Run on semver tag pushes: `on: { push: { tags: ['v[0-9]+.[0-9]+.[0-9]+', 'v[0-9]+.[0-9]+.[0-9]+-*'] } }`
- NO concurrency control (never cancel releases)

**Global permissions:**
- Set to read-only: `permissions: { contents: read }`

**Job 1: build**
- Name: "Build distributions"
- Runner: ubuntu-latest
- Timeout: 10 minutes
- Steps:
  1. Checkout with full history: `actions/checkout@v6` with `fetch-depth: 0`
  2. Setup uv: `astral-sh/setup-uv@v7` with caching
  3. Sync dependencies: `uv sync --locked --dev`
  4. Build wheel and sdist: `uv build`
  5. Upload artifacts: `actions/upload-artifact@v4` with name "dist" and path "dist/" and retention-days: 7

**Job 2: validate**
- Name: "Validate distributions"
- Depends on: build (`needs: [build]`)
- Runner: ubuntu-latest
- Timeout: 10 minutes per Python version
- Strategy: matrix with python-version ["3.11", "3.14"], fail-fast: false
- Steps:
  1. Setup uv: `astral-sh/setup-uv@v7` with python-version from matrix
  2. Download artifacts: `actions/download-artifact@v4` with name "dist" and path "dist/"
  3. Test wheel installation:
     ```bash
     uv venv --python ${{ matrix.python-version }} test-env-wheel
     uv pip install --python test-env-wheel/bin/python dist/*.whl
     test-env-wheel/bin/python -c "import cubano; print(f'cubano version: {cubano.__version__}')"
     test-env-wheel/bin/python -c "from cubano import SemanticView, Query, register, Row; print('Public API imports OK')"
     ```
  4. Test sdist installation:
     ```bash
     uv venv --python ${{ matrix.python-version }} test-env-sdist
     uv pip install --python test-env-sdist/bin/python dist/*.tar.gz
     test-env-sdist/bin/python -c "import cubano; print(f'cubano version: {cubano.__version__}')"
     ```
  5. Check py.typed marker in wheel:
     ```bash
     unzip -l dist/*.whl | grep "cubano/py.typed" || (echo "ERROR: py.typed marker missing" && exit 1)
     ```
  6. Inspect wheel metadata:
     ```bash
     unzip -p dist/*.whl "*/METADATA" | head -20
     ```

**Job 3: changelog**
- Name: "Generate changelog"
- Depends on: build (`needs: [build]`)
- Runner: ubuntu-latest
- Timeout: 5 minutes
- Steps:
  1. Checkout with full history: `actions/checkout@v6` with `fetch-depth: 0`
  2. Install git-cliff:
     ```bash
     curl -L https://github.com/orhun/git-cliff/releases/download/v2.7.0/git-cliff-2.7.0-x86_64-unknown-linux-gnu.tar.gz | tar xz -C /usr/local/bin
     ```
  3. Generate changelog: `git-cliff --latest --output CHANGELOG.md`
  4. Upload changelog: `actions/upload-artifact@v4` with name "changelog" and path "CHANGELOG.md"

**Job 4: publish**
- Name: "Publish to PyPI"
- Depends on: validate and changelog (`needs: [validate, changelog]`)
- Runner: ubuntu-latest
- Timeout: 5 minutes
- Environment: `environment: { name: pypi, url: https://pypi.org/p/cubano }`
- Permissions: `permissions: { id-token: write, contents: read }`
- Steps:
  1. Download artifacts: `actions/download-artifact@v4` with name "dist" and path "dist/"
  2. Publish to PyPI: `pypa/gh-action-pypi-publish@release/v1` with no additional config (Trusted Publishing via OIDC)

**CRITICAL:**
- Validation job MUST complete successfully before publish runs
- Use `needs: [validate, changelog]` to create hard block
- Trusted Publishing requires `id-token: write` permission
- Full git history needed for changelog generation (`fetch-depth: 0`)

Per user decisions:
- Release on semver tags (v1.2.3)
- Validate both wheel and sdist in isolated venvs
- Hard block on failed validation
- Auto-generate changelog using conventional commits

<task type="auto">
  <name>Create git-cliff configuration for changelog generation</name>
  <files>.cliff.toml</files>
  <action>
Create git-cliff configuration file `.cliff.toml` for changelog generation from conventional commits:

```toml
[changelog]
# Changelog header
header = """
# Changelog\n
All notable changes to this project will be documented in this file.\n
"""
# Template for the changelog body
body = """
{% for group, commits in commits | group_by(attribute="group") %}
    ### {{ group | upper_first }}
    {% for commit in commits %}
        - {{ commit.message | upper_first }}\
    {% endfor %}
{% endfor %}\n
"""
# Remove leading and trailing whitespace
trim = true

[git]
# Parse commits based on conventional commits specification
conventional_commits = true
# Filter out commits that are not conventional
filter_unconventional = true
# Group commits by type
commit_parsers = [
    { message = "^feat", group = "Features" },
    { message = "^fix", group = "Bug Fixes" },
    { message = "^docs", group = "Documentation" },
    { message = "^perf", group = "Performance" },
    { message = "^refactor", group = "Refactoring" },
    { message = "^style", group = "Styling" },
    { message = "^test", group = "Testing" },
    { message = "^chore", group = "Miscellaneous" },
]
# Sort commits by type
sort_commits = "oldest"
```

This configuration:
- Parses conventional commit messages (feat:, fix:, docs:, etc.)
- Groups changes by type (Features, Bug Fixes, etc.)
- Filters out non-conventional commits
- Generates clean, categorized changelog entries

Per user decision: Auto-generate changelog on release using conventional commits.
