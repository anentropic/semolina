<task type="auto">
  <name>Create CI workflow with 4 quality check jobs</name>
  <files>.github/workflows/ci.yml</files>
  <action>
Create GitHub Actions workflow `.github/workflows/ci.yml` with the following configuration:

**Triggers:**
- Run on all pushes to any branch: `on: [push]`
- Enable concurrency control: `concurrency: { group: ci-${{ github.ref }}, cancel-in-progress: true }`

**Global permissions:**
- Set to read-only by default: `permissions: { contents: read }`

**Jobs (4 required checks):**

1. **typecheck job:**
   - Name: "Type check with basedpyright"
   - Runner: ubuntu-latest
   - Timeout: 5 minutes
   - Steps: checkout@v6, setup-uv@v7 with enable-cache and cache-dependency-glob="uv.lock", sync dependencies with `uv sync --locked --dev`, run `uv run basedpyright`
   - No special output format needed (basedpyright auto-detects GitHub Actions)

2. **lint job:**
   - Name: "Lint with ruff"
   - Runner: ubuntu-latest
   - Timeout: 5 minutes
   - Steps: checkout@v6, setup-uv@v7 with caching, sync deps, run `uv run ruff check --output-format=github`
   - Use `--output-format=github` for inline annotations

3. **format job:**
   - Name: "Format check with ruff"
   - Runner: ubuntu-latest
   - Timeout: 5 minutes
   - Steps: checkout@v6, setup-uv@v7 with caching, sync deps, run `uv run ruff format --check`
   - Use `--check` flag to verify formatting without modifying files

4. **test job:**
   - Name: "Test with pytest"
   - Runner: ubuntu-latest
   - Timeout: 10 minutes
   - Strategy: matrix with python-version ["3.11", "3.14"], fail-fast: false
   - Steps: checkout@v6, setup-uv@v7 with caching and python-version from matrix, sync deps, run `uv run pytest tests/`
   - Add cache pruning step at end: `if: always()` with `uv cache prune --ci`

**CRITICAL:**
- Job names (typecheck, lint, format, test) MUST be unique and match exactly for branch protection status checks
- Use `astral-sh/setup-uv@v7` (official uv action)
- Use `uv sync --locked --dev` for reproducible installs
- All jobs run in parallel (no dependencies)

Per user decision: CI runs on all branches (every push triggers CI), no Slack/email notifications (GitHub checks only).
