<task type="auto">
  <name>Create PR workflow with coverage reporting</name>
  <files>.github/workflows/pr.yml</files>
  <action>
Create GitHub Actions workflow `.github/workflows/pr.yml` with the following configuration:

**Triggers:**
- Run only on pull_request events: `on: [pull_request]`
- Enable concurrency control: `concurrency: { group: pr-${{ github.ref }}, cancel-in-progress: true }`

**Permissions:**
- Grant write access for PR comments: `permissions: { pull-requests: write, contents: read }`

**Job: coverage**
- Name: "Report test coverage"
- Runner: ubuntu-latest
- Timeout: 10 minutes
- Steps:
  1. Checkout code: `actions/checkout@v6`
  2. Setup uv: `astral-sh/setup-uv@v7` with enable-cache and cache-dependency-glob="uv.lock"
  3. Sync dependencies: `uv sync --locked --dev`
  4. Run tests with coverage: `uv run pytest --cov=src/cubano --cov-report=xml --cov-report=term`
  5. Post coverage comment: `MishaKav/pytest-coverage-comment@v1.1.51` with:
     - `pytest-coverage-path: coverage.xml`
     - `github-token: ${{ secrets.GITHUB_TOKEN }}`
  6. Cache pruning: `if: always()` with `uv cache prune --ci`

**Key points:**
- Use `--cov-report=xml` to generate coverage.xml for the action
- Use `--cov-report=term` to show coverage in job logs
- MishaKav action automatically updates existing comments (no duplicates)
- No minimum threshold enforcement per user decision

Per user decision: Coverage tracked but not enforced (no minimum threshold).
  </action>
