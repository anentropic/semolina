    1. In src/semolina/registry.py, update reset() to try close_pool() first, fall back to pool.close():
       ```python
       def reset() -> None:
           """
           Clear all registered pools and engines (for testing only).

           Uses ``close_pool()`` from adbc-poolhouse for proper ADBC resource
           cleanup. Falls back to ``pool.close()`` for pools without an ADBC
           source connection (e.g. MockPool).
           """
           for pool, _dialect in _pools.values():
               with contextlib.suppress(Exception):
                   if hasattr(pool, "_adbc_source"):
                       from adbc_poolhouse import close_pool
                       close_pool(pool)
                   else:
                       pool.close()
           _pools.clear()
           _engines.clear()
       ```
       Note: The `from adbc_poolhouse import close_pool` is a lazy import inside the function body. This is intentional -- reset() is only called during testing, and using a lazy import here avoids changing the module-level imports of registry.py (which does not otherwise import adbc_poolhouse). The hasattr check ensures MockPool (which has no _adbc_source) still gets pool.close().

    2. In tests/unit/test_registry.py, add a test verifying close_pool behavior:

       - Ensure the existing test_reset_calls_close_on_pools still passes (MockPool-like objects without _adbc_source use pool.close()).
