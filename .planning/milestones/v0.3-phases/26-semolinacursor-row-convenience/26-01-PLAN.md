<objective>
Create the SemolinaCursor class and make MockCursor fully DBAPI 2.0 compliant.

Purpose: SemolinaCursor is the new return type of .execute() (replacing Result). It wraps any DBAPI 2.0 cursor via delegation and adds fetchall_rows(), fetchmany_rows(), fetchone_row() convenience methods that convert raw tuples to Row objects. MockCursor needs execute(sql, params) and fetchmany() to be a standard DBAPI 2.0 cursor that SemolinaCursor can delegate to.

Output: src/semolina/cursor.py (new), updated src/semolina/pool.py, tests/unit/test_cursor.py (new)
</objective>

    **Create src/semolina/cursor.py:**

    New file with SemolinaCursor class. Use `from __future__ import annotations`. Import `Any` from typing and `Row` from `.results`.

    Class SemolinaCursor with:
    - `__init__(self, cursor: Any, conn: Any, pool: Any) -> None` storing as `self._cursor`, `self._conn`, `self._pool`
    - `_column_names(self) -> list[str]` extracting `[d[0] for d in self._cursor.description]` or empty list if description is None
    - `fetchall_rows(self) -> list[Row]` calling `self._cursor.fetchall()`, converting via `Row(dict(zip(columns, row, strict=True)))`
    - `fetchone_row(self) -> Row | None` calling `self._cursor.fetchone()`, returning None if raw is None, else Row conversion
    - `fetchmany_rows(self, size: int = 1) -> list[Row]` calling `self._cursor.fetchmany(size)`, converting each row
    - `fetchall(self) -> list[tuple[Any, ...]]` passthrough to `self._cursor.fetchall()`
    - `fetchone(self) -> tuple[Any, ...] | None` passthrough to `self._cursor.fetchone()`
    - `fetchmany(self, size: int = 1) -> list[tuple[Any, ...]]` passthrough to `self._cursor.fetchmany(size)`
    - `description` property passthrough to `self._cursor.description` with return type `list[tuple[Any, ...]] | None`
    - `rowcount` property passthrough to `self._cursor.rowcount` with return type `int`
    - `close(self) -> None` calling `self._cursor.close()` then `self._conn.close()`
    - `__enter__(self) -> SemolinaCursor` returning self
    - `__exit__(self, *exc: Any) -> None` calling `self.close()`
    - `__repr__(self) -> str` showing `<SemolinaCursor columns=[...] open>` or `<SemolinaCursor closed>` (track closed state in close())

    Docstring style: opening/closing `"""` on own lines for multi-line, Google style. D213 (summary on second line after opening quotes). Line length 100 chars.

    **Update src/semolina/pool.py -- add two methods to MockCursor:**

    1. `execute(self, sql: str, params: Any = None) -> None` -- DBAPI 2.0 execute. Extract view name from SQL using regex `FROM\s+"?(\w+)"?` (case-insensitive). Look up that view in self._fixtures. If found, populate self._columns, self._rows (as tuples), self._description. If not found or no fixtures, set empty results. Reset self._pos = 0.

    2. `fetchmany(self, size: int = 1) -> list[tuple[Any, ...]]` -- Return up to `size` rows from self._pos, advance self._pos by the number of rows returned. Implementation: `result = self._rows[self._pos : self._pos + size]; self._pos += len(result); return result`.

    The execute method should use `import re` at the module top (add it). The regex pattern to extract view name from SQL: `re.search(r'FROM\s+"?(\w+)"?', sql, re.IGNORECASE)`. If match, view_name = match.group(1). Then same logic as _execute_query for populating rows/columns/description from fixtures.
