# Requirements: Semolina

**Defined:** 2026-03-16
**Core Value:** A single, Pythonic query API that works identically across Snowflake and Databricks semantic views, with typed models, IDE autocomplete, and backend-agnostic code.

## v0.3 Requirements

Requirements for Arrow & Connection Layer milestone. Each maps to roadmap phases.

### Connection Layer

- [x] **CONN-01**: User can register a pool with a dialect tag via `register("default", pool, dialect="snowflake")`
- [x] **CONN-02**: User can select a registered pool via `.using("name")` on queries
- [x] **CONN-03**: Dialect enum determines SQL generation (AGG vs MEASURE, placeholder style)
- [x] **CONN-04**: User can test without a warehouse using MockPool with in-memory Arrow data

### Cursor & Results

- [x] **CURS-01**: `.execute()` returns a SemolinaCursor that subclasses the ADBC cursor
- [x] **CURS-02**: SemolinaCursor adds `fetchall_rows()` returning `list[Row]`
- [x] **CURS-03**: SemolinaCursor adds `fetchmany_rows(size)` returning `list[Row]`
- [x] **CURS-04**: SemolinaCursor adds `fetchone_row()` returning `Row | None`
- [x] **CURS-05**: Row retains attribute access (`row.revenue`) and dict access (`row["revenue"]`)

### Configuration

- [x] **CONF-01**: `.semolina.toml` connection sections have a backend sub-section (snowflake/databricks) determining config class
- [x] **CONF-02**: `.semolina.toml` sections load into adbc-poolhouse config classes via TomlSettingsSource
- [x] **CONF-03**: User can create a pool via `pool_from_config(connection="conn1")` with default `.semolina.toml` path

### Query API

- [x] **QAPI-01**: `query(metrics=[...], dimensions=[...])` shorthand accepted as keyword args
- [x] **QAPI-02**: Builder methods (`.metrics()`, `.dimensions()`) are additive with args passed to `query()`

### Documentation

- [x] **DOCS-01**: Tutorials updated for pool registration and SemolinaCursor API
- [x] **DOCS-02**: How-to guides updated for .semolina.toml connection config and pool_from_config()
- [x] **DOCS-03**: How-to guides updated for Arrow fetch methods and Row convenience methods
- [x] **DOCS-04**: API reference generated for new public API (SemolinaCursor, Dialect, pool_from_config, register with dialect)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Multi-view join API | Complex feature, requires extensive design work |
| Window functions (ROW_NUMBER, LAG, etc.) | SQL complexity beyond semantic view scope |
| HAVING clause for metric filtering | Evaluate after core query API stabilizes |
| SEMANTIC_VIEW() clause syntax for Snowflake | Using standard SQL instead |
| dbt manifest → Semolina model codegen | Focus is warehouse-direct introspection |
| Connection pooling logic in Semolina | adbc-poolhouse handles this |
