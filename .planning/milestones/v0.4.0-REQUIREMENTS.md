**Defined:** 2026-04-19
**Core Value:** A single, Pythonic query API that works identically across Snowflake, Databricks, and DuckDB semantic views, with typed models, IDE autocomplete, and backend-agnostic code.

## Out of Scope (final)

| Feature | Reason |

| Narwhals integration | `.fetch_arrow_table()` + user-side conversion is sufficient; narwhals adds dep weight for marginal DX gain |

| Async query execution | Architecture change, evaluate separately |

## Notes on Changed Requirements

- **ARROW-02 naming.** The original requirement text named the API `to_arrow()`. The shipped public API is `fetch_arrow_table()`, matching the underlying ADBC convention. Documentation and traceability now reflect the shipped name.
- **ARROW-01 surface.** No standalone `[arrow]` extra was shipped. Each backend extra already includes pyarrow as a transitive ADBC dependency, so a separate `[arrow]` extra would have been redundant.
- **ARROW-03 error path.** The "clear ImportError if pyarrow missing" branch is effectively unreachable in normal use because every supported ADBC backend hard-depends on pyarrow. Behaviour documented as intended.
