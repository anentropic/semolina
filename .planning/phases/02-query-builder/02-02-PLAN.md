<feature>
  <name>Immutable Query Builder</name>
  <files>src/cubano/query.py, tests/test_query.py, src/cubano/__init__.py</files>
  <behavior>
    Query is a frozen dataclass. Each method returns a new instance via dataclasses.replace(). Fields are stored as tuples (immutable).

    Cases — .metrics():
    - Query().metrics(Sales.revenue) -> Query with _metrics=(Sales.revenue,)
    - Query().metrics(Sales.revenue, Sales.cost) -> Query with _metrics=(Sales.revenue, Sales.cost)
    - Query().metrics(Sales.country) -> TypeError ("Did you mean .dimensions()?")
    - Query().metrics() -> ValueError ("At least one metric must be provided")

    Cases — .dimensions():
    - Query().dimensions(Sales.country) -> Query with _dimensions=(Sales.country,)
    - Query().dimensions(Sales.country, Sales.unit_price) -> accepts both Dimension and Fact
    - Query().dimensions(Sales.revenue) -> TypeError ("Did you mean .metrics()?")
    - Query().dimensions() -> ValueError ("At least one dimension must be provided")

    Cases — .filter():
    - q.filter(Q(country='US')) -> Query with _filters=Q(country='US')
    - q.filter(Q(a=1)).filter(Q(b=2)) -> ANDs filters together
    - q.filter("not a Q") -> TypeError

    Cases — .order_by():
    - q.order_by(Sales.revenue) -> Query with _order_by_fields=(Sales.revenue,)
    - q.order_by(Sales.revenue, Sales.country) -> multiple fields
    - q.order_by("revenue") -> TypeError
    - q.order_by() -> ValueError

    Cases — .limit():
    - q.limit(100) -> Query with _limit_value=100
    - q.limit(0) -> ValueError ("requires positive integer")
    - q.limit(-5) -> ValueError
    - q.limit(3.14) -> TypeError ("requires int")

    Cases — immutability (QRY-07):
    - q1 = Query(); q2 = q1.metrics(Sales.revenue) -> q1._metrics == (), q2._metrics == (Sales.revenue,)
    - Chained: Query().metrics(A).dimensions(B).filter(Q(c='d')).order_by(A).limit(10) -> all fields populated
    - Original query unchanged after any method call

    Cases — _validate_for_execution():
    - Query()._validate_for_execution() -> ValueError ("must select at least one metric or dimension")
    - Query().metrics(Sales.revenue)._validate_for_execution() -> no error
    - Query().dimensions(Sales.country)._validate_for_execution() -> no error

    Cases — accumulation:
    - q.metrics(A).metrics(B) -> _metrics=(A, B) (appends, does not replace)
    - q.dimensions(A).dimensions(B) -> _dimensions=(A, B)
    - q.order_by(A).order_by(B) -> _order_by_fields=(A, B)

    Key implementation details from research:
    - @dataclass(frozen=True) for immutability
    - dataclasses.replace() for creating modified copies
    - Tuple storage (not lists) for all field collections
    - isinstance() checks in every method for runtime type safety
    - Helpful error messages with "Did you mean...?" guidance
    - _validate_for_execution() called by to_sql() and fetch() (Phase 3/4 stubs)
    - to_sql() and fetch() raise NotImplementedError for now (implemented in later phases)
  </behavior>
  <implementation>
    Follow the Query class skeleton from 02-RESEARCH.md "Complete Query Class Skeleton" section.

    The frozen dataclass needs:
    1. _metrics: tuple[Metric, ...] = field(default_factory=tuple)
    2. _dimensions: tuple[Dimension | Fact, ...] = field(default_factory=tuple)
    3. _filters: Q | None = None
    4. _order_by_fields: tuple[Field, ...] = field(default_factory=tuple)
    5. _limit_value: int | None = None

    Methods (each returns new Query via replace()):
    6. metrics(*fields) - validate Metric types, append to _metrics
    7. dimensions(*fields) - validate Dimension|Fact types, append to _dimensions
    8. filter(condition) - validate Q type, AND with existing _filters
    9. order_by(*fields) - validate Field types, append to _order_by_fields
    10. limit(n) - validate positive int, set _limit_value
    11. _validate_for_execution() - check at least one metric or dimension
    12. to_sql() -> str - stub that validates then raises NotImplementedError
    13. fetch() -> list - stub that validates then raises NotImplementedError

    Update __init__.py to export Query and Q:
    - from .query import Query
    - from .filters import Q
    - Add both to __all__
  </implementation>
</feature>
