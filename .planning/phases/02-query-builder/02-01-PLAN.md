<feature>
  <name>Q-Object Filter Composition</name>
  <files>src/cubano/filters.py, tests/test_filters.py</files>
  <behavior>
    Q-objects encapsulate filter conditions and compose via boolean operators into tree structures.

    Cases:
    - Q(country='US') -> leaf node, children=[('country', 'US')], connector=AND, negated=False
    - Q(country='US', region='West') -> leaf node, children=[('country', 'US'), ('region', 'West')], connector=AND
    - Q(a=1) | Q(b=2) -> branch node, children=[Q(a=1), Q(b=2)], connector=OR
    - Q(a=1) & Q(b=2) -> branch node, children=[Q(a=1), Q(b=2)], connector=AND
    - ~Q(a=1) -> branch node, children=[Q(a=1)], negated=True
    - (Q(a=1) | Q(b=2)) & Q(c=3) -> AND node with [OR-node, Q(c=3)] children
    - Q() with no kwargs -> empty leaf, children=[]
    - Q(a=1) | "not a Q" -> TypeError
    - Q(a=1) & 42 -> TypeError
    - repr(Q(country='US')) -> "Q(country='US')"
    - repr(Q(a=1) | Q(b=2)) -> "(Q(a=1) | Q(b=2))"
    - repr(~Q(a=1)) -> "~(Q(a=1))"

    Structure: Q is NOT a dataclass (custom __init__ needed). Uses tree structure with:
    - children: list of child Q objects or (key, value) tuples
    - connector: "AND" or "OR" class constant
    - negated: bool
    - AND/OR as class-level string constants

    Key implementation details from research:
    - Use Q.__new__(Q) in _combine and __invert__ to bypass __init__
    - Sort kwargs.items() in __init__ for consistent ordering
    - _combine creates new Q with both operands as children
    - __invert__ wraps self in a new Q with negated=True
    - TypeError if combining Q with non-Q
  </behavior>
  <implementation>
    Follow the Q-object implementation from 02-RESEARCH.md "Complete Q-Object Implementation" section.

    The class needs:
    1. Class constants: AND = "AND", OR = "OR"
    2. __init__(**kwargs) - create leaf node from keyword conditions
    3. _combine(other, conn) - internal method to merge two Q objects
    4. __or__(other) - calls _combine with OR
    5. __and__(other) - calls _combine with AND
    6. __invert__() - wrap in negated Q
    7. __repr__() - readable string for debugging (distinguish leaf vs branch)
    8. __bool__() - return True if Q has children (useful for truthiness checks)

    Do NOT use @dataclass decorator. Q needs custom __init__ and mutable internals for tree building.
  </implementation>
</feature>
