Add a Result class to results.py that wraps List[Row] and enables future helper methods:

1. Add Result class to results.py (at the end, after Row class):
   ```python
   class Result:
       """
       Typed result wrapper for query execution.

       Result wraps a list of Row objects and provides helper methods for
       data access and transformation. Supports iteration, indexing, and
       attribute/dict-style row access.

       Example:
           >>> result = Users.query().metrics(Users.revenue).execute()
           >>> len(result)
           10
           >>> for row in result:
           ...     print(row.revenue)
           >>> result[0].revenue
           1000
       """

       def __init__(self, rows: list[Row]) -> None:
           """
           Initialize Result with row data.

           Args:
               rows: List of Row objects from query execution
           """
           self.rows: list[Row] = rows

       def __len__(self) -> int:
           """Return number of rows in result."""
           return len(self.rows)

       def __iter__(self):
           """Iterate over Row objects."""
           return iter(self.rows)

       def __getitem__(self, index: int) -> Row:
           """
           Get row by index.

           Args:
               index: Row index (0-based)

           Returns:
               Row object

           Raises:
               IndexError: If index out of range
           """
           return self.rows[index]

       def __repr__(self) -> str:
           """Return string representation."""
           return f"Result({len(self.rows)} rows)"

       def __bool__(self) -> bool:
           """Return True if result has any rows."""
           return len(self.rows) > 0
   ```

2. Add type annotations to Result class:
   - Use list[Row] (modern style, not List[Row])
   - Return type of __iter__ is Iterator[Row]
   - Return type of __getitem__ is Row

   ```python
   def execute(self) -> Result:
       """
       Execute query and return results immediately (eager).

       Wrapper around .fetch() that returns results in a Result object
       instead of a bare list. Result preserves row access patterns and
       enables future helper methods.

       Returns:
           Result object wrapping List[Row]

       Raises:
           Exception: If query execution fails (warehouse connection, SQL error, etc.)

       Example:
           >>> result = (Users.query()
           ...     .metrics(Users.revenue)
           ...     .dimensions(Users.country)
           ...     .execute())
           >>> len(result)
           5
           >>> for row in result:
           ...     print(row.country, row.revenue)
       """
       rows = self.fetch()
       return Result(rows)
   ```

3. Update Query docstring to mention both .fetch() and .execute():

- .execute() is a thin wrapper that calls .fetch() and wraps in Result
- Eager execution happens at .fetch() time, not at .execute() time
- Result object is created fresh for each .execute() call (no caching)
