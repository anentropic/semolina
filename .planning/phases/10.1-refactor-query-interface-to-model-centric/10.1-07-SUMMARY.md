# Phase 10.1 Plan 07: Remove Old Query API & Make Constructor Private

## Objective

Complete Phase 10.1 refactoring by eliminating backward compatibility code and enforcing model-centric query construction as the only public entry point.

## What Was Built

Removed deprecated procedural Query API and made the Query class private:

1. **Removed Query.filter()** - Was deprecated in 10.1-02, completely deleted
   - Removed ~30 lines of method and docstring
   - Tests updated to use .where() instead
   - No references remain in source code

2. **Removed Query.fetch()** - Was internal/backward compatibility method
   - Functionality merged into Query.execute()
   - execute() now handles full pipeline: validation → engine resolution → execution → Result wrapping
   - Tests converted to use execute() and verify Result object behavior

3. **Made Query Private** - Renamed to _Query
   - Query class renamed from `Query` to `_Query`
   - Removed from public API exports in __init__.py
   - `from cubano import Query` now raises ImportError (intended behavior)
   - All internal code updated to use _Query
   - Model.query() remains the only documented entry point

4. **Updated All Imports** - Internal consistency
   - models.py, engines (base, snowflake, databricks, sql, mock), tests updated
   - TYPE_CHECKING imports use _Query
   - Method signatures use _Query in type annotations
   - Docstring examples show Model.query() as public API

5. **Updated Test Suite** - 433 tests pass
   - test_query.py converted to use _Query internally
   - .filter() calls replaced with .where()
   - .fetch() calls replaced with .execute()
   - TestQueryFetch class renamed to test execute() returning Result
   - test_filter_rejects_non_q removed (type system handles validation)
