## Test Patterns for Future Development

### Model Definition Pattern

```python
class Sales(SemanticView, view="sales"):
    revenue = Metric()
    country = Dimension()
```

### Query Construction Pattern

```python
# Using new model-centric API
result = (Sales.query()
    .metrics(Sales.revenue)
    .dimensions(Sales.country)
    .where(Sales.country == 'US')
    .where(Sales.revenue > 1000)
    .order_by(Sales.revenue.desc())
    .limit(100)
    .execute())
```

### Field Operator Pattern

```python
# Equality
Sales.country == 'US'  # → Q(country='US')

# Comparison
Sales.revenue > 1000   # → Q(revenue__gt=1000)

# Composition
(Sales.country == 'US') | (Sales.country == 'CA')  # → Q with OR

# In where()
query.where((Sales.country == 'US') | (Sales.country == 'CA'))
```

### Result Access Patterns

```python
result = Sales.query().metrics(Sales.revenue).execute()

# Length and bool
if result:
    print(f"Got {len(result)} rows")

# Iteration
for row in result:
    print(row.revenue)

# Indexing
first_revenue = result[0].revenue

# Row access patterns
result[0].revenue        # attribute access
result[0]['revenue']     # dict-style access
```

## Migration Path for External Codebases

For users with existing code using procedural Query() API:

```python
# OLD API (still works, marked as internal):
results = Query().metrics(Users.revenue).filter(Q(country='US')).fetch()

# NEW API (model-centric, recommended):
results = Users.query().metrics(Users.revenue).where(Users.country == 'US').execute()
```

Key differences:
- Model.query() instead of Query()
- .where() instead of .filter()
- Field operators instead of Q objects: field == value instead of Q(field=value)
- .execute() instead of .fetch() (returns Result instead of list)
- Results in Result object (supports iteration, indexing, bool)
