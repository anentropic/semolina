  <name>Task 2: Rewrite cli/codegen.py with reverse codegen CLI</name>
  <files>src/cubano/cli/codegen.py</files>
  <action>
    Replace the entire content of src/cubano/cli/codegen.py with the new reverse codegen command.

    The new CLI signature:
    ```python
    def codegen(
        views: Annotated[
            list[str],
            typer.Argument(help="Schema-qualified view names (e.g. my_schema.my_view)")
        ],
        backend: Annotated[
            str,
            typer.Option("--backend", "-b", help="Backend: snowflake, databricks, or dotted.path.ClassName")
        ],
    ) -> None:
        """Introspect warehouse semantic views and generate Cubano model classes."""
    ```

    Key implementation details:
    - Remove the old Backend StrEnum (--backend is now plain str, not an enum)
    - Remove the old --output, --verbose, --no-comments flags (deferred per locked decisions in CONTEXT.md)
    - No import of resolve_input_paths or generate_sql_for_files (old forward codegen removed)
    - Keep _stderr = Console(file=sys.stderr, stderr=True) for error/warning output
    - Keep typer.echo() for Python source output (so CliRunner captures it in tests)

    _resolve_backend(backend_spec: str) -> Engine module-level private function:
    - if backend_spec == "snowflake":
        from cubano.testing.credentials import SnowflakeCredentials
        from cubano.engines.snowflake import SnowflakeEngine
        creds = SnowflakeCredentials.load()
        return SnowflakeEngine(**creds.model_dump())
    - elif backend_spec == "databricks":
        from cubano.testing.credentials import DatabricksCredentials
        from cubano.engines.databricks import DatabricksEngine
        creds = DatabricksCredentials.load()
        return DatabricksEngine(**creds.model_dump())
    - else (dotted import path):
        module_path, _, class_name = backend_spec.rpartition(".")
        if not module_path:
            raise typer.BadParameter(f"Unknown backend {backend_spec!r}. Use 'snowflake', 'databricks', or a dotted import path.")
        try:
            import importlib
            module = importlib.import_module(module_path)
            cls = getattr(module, class_name)
            return cls()
        except (ImportError, AttributeError) as e:
            raise typer.BadParameter(f"Cannot import backend {backend_spec!r}: {e}") from e

    codegen() command body:
    1. engine = _resolve_backend(backend) — wrap in try/except typer.BadParameter; print to stderr, exit 1
    2. introspected_views = []
    3. For each view_name in views:
         try:
             introspected = engine.introspect(view_name)
             introspected_views.append(introspected)
         except RuntimeError as e:
             _stderr.print(f"[bold red]Error:[/bold red] {e}")
             raise typer.Exit(code=1) from e
    4. from cubano.codegen.python_renderer import render_and_format
       source = render_and_format(introspected_views)
    5. typer.echo(source)

    Preserve the module-level _stderr Console for errors outside the command function.
    Remove all imports that no longer apply (StrEnum, Path, resolve_input_paths, etc.).
    Use `from __future__ import annotations` at top.

    basedpyright note: The Engine return type from _resolve_backend needs proper annotation.
    Use `from __future__ import annotations` + `Engine` type hint. Import Engine from cubano.engines.base.
    The `**creds.model_dump()` call: model_dump() returns dict[str, Any]; Engine.__init__ accepts **Any — should typecheck cleanly.
