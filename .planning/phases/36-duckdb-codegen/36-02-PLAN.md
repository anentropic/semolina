    def test_duckdb_backend_database_env_var(self) -> None:

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI argument -> _resolve_backend | --backend and --database values come from user CLI input |
| DUCKDB_DATABASE env var -> database path | Environment variable read by Typer envvar mechanism |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-36-04 | Tampering | --database CLI option | accept | Database path is provided by the user running the CLI on their own machine. DuckDB validates file paths internally. No path traversal risk beyond what the user already has filesystem access to. |
| T-36-05 | Tampering | --backend option | mitigate | Unknown backend values are rejected with typer.BadParameter. The "duckdb" alias only accepts paths via --database. Dotted import paths are validated by Python's import machinery. |
| T-36-06 | Information Disclosure | DUCKDB_DATABASE env var | accept | Environment variable is read by the CLI process only. Value is a file path, not a secret. No logging of the value occurs. |
</threat_model>
