    Update the "Run unit tests" step in `.github/workflows/ci.yml` to add `--snapshot-warn-unused`.

    Current command:
    ```yaml
    - name: Run unit tests
      run: uv run pytest tests/ -n auto -v
    ```

    Updated command:
    ```yaml
    - name: Run unit tests
      run: uv run pytest tests/ -n auto -v --snapshot-warn-unused
    ```

    **Why `--snapshot-warn-unused` not `--snapshot-default-extension`:**
    - `--snapshot-warn-unused`: syrupy warns when `.ambr` files contain snapshots with no matching test (stale), but does NOT fail the build. This is the right CI balance — inform the developer without blocking on cleanup.
    - Without this flag: stale snapshots accumulate silently, leading to `.ambr` files with orphaned entries over time.
    - With `--snapshot-update` (recording mode, local only): syrupy auto-deletes stale entries. No CI action needed for this.

    **No other CI changes are needed:**
    - Snapshot tests in `tests/test_snapshot_queries.py` run automatically as part of `pytest tests/`
    - All 12 variants ([snowflake_engine] and [databricks_engine] for each of 6 tests) use MockEngine in replay mode — no warehouse credentials needed
    - The parametrized `backend_engine` fixture drives both backends; CI never needs SNOWFLAKE_* or DATABRICKS_* env vars for snapshot replay
    - No new CI job, step, or secret is required

    After editing ci.yml, validate the YAML syntax:
