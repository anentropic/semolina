**`docs/guides/queries.md`** (DOCS-03)

Cover all 8 Query methods:
1. `.metrics(*fields)` — selecting aggregatable measures; TypeError if non-Metric
2. `.dimensions(*fields)` — grouping by Dimension/Fact fields; TypeError if non-Dimension/Fact
3. `.filter(condition)` — Q object condition; multiple calls are ANDed
4. `.order_by(*fields)` — bare Field (ascending) or OrderTerm for direction
5. `.limit(n)` — positive integer only
6. `.using(name)` — lazy engine resolution at fetch time
7. `.fetch()` — executes and returns list of Row objects
8. `.to_sql()` — generates SQL using MockDialect for inspection

Show immutable chaining pattern explicitly (this is a key concept):
```python
# Each method returns a NEW Query — queries are reusable
base = Query().metrics(Sales.revenue).dimensions(Sales.country)
us_only = base.filter(Q(country="US"))
top_10 = base.limit(10)  # base is unchanged
```

Include a `!!! tip "Inspect SQL before executing"` showing `.to_sql()` usage.

**`docs/guides/filtering.md`** (DOCS-04)

Cover:
- Basic equality: `Q(country="US")`
- Lookup expressions: `Q(revenue__gt=1000)`, `Q(revenue__gte=500)`, `Q(country__contains="US")` — document all supported lookups (gt, gte, lt, lte, contains, startswith, endswith, in, isnull)
- OR composition: `Q(country="US") | Q(country="CA")`
- AND composition: `Q(country="US") & Q(revenue__gt=500)`
- NOT negation: `~Q(country="US")`
- Complex nesting with parentheses: `(Q(country="US") | Q(country="CA")) & ~Q(revenue__lt=100)`
- Multiple `.filter()` calls (each call ANDs with the previous)
- Operator precedence WARNING as a `!!! warning` admonition: `&` binds tighter than `|` — always parenthesize OR combinations

**`docs/guides/backends/overview.md`** (DOCS-05)

Backend comparison table:
| Feature | Snowflake | Databricks |
|---------|-----------|------------|
| Engine class | `SnowflakeEngine` | `DatabricksEngine` |
| Install extra | `cubano[snowflake]` | `cubano[databricks]` |
| Metric SQL function | `AGG(revenue)` | `MEASURE(revenue)` |
| Identifier quoting | Double quotes `"name"` | Backticks `` `name` `` |
| View type | CREATE SEMANTIC VIEW | Metric Views |
| Unity Catalog | N/A | Three-part names supported |

Key message: The Query API is IDENTICAL for both backends. Only the engine changes.

Show the pattern:
```python
# Snowflake
from cubano.engines.snowflake import SnowflakeEngine
register("default", SnowflakeEngine(**snowflake_params))

# Databricks — same query, different engine
from cubano.engines.databricks import DatabricksEngine
register("default", DatabricksEngine(**databricks_params))

# Query is identical
results = Query().metrics(Sales.revenue).dimensions(Sales.country).fetch()
```
