**Snowflake tab (replace lines 37-44):** Replace with valid `CREATE OR REPLACE SEMANTIC VIEW` syntax:

```sql
CREATE OR REPLACE SEMANTIC VIEW sales
  TABLES (
    s AS source_table PRIMARY KEY (id)
  )
  DIMENSIONS (
    s.country AS country,
    s.region AS region
  )
  METRICS (
    s.revenue AS SUM(revenue),
    s.cost AS SUM(cost)
  )
;
```

Key syntax rules (from 18-RESEARCH.md):
- TABLES clause is mandatory, uses `alias AS table_name PRIMARY KEY (col)` format
- DIMENSIONS use `table_alias.name AS expression` format
- METRICS use `table_alias.name AS AGG_FUNCTION(column)` format (SUM, not AGG)
- Clause order is fixed: TABLES, DIMENSIONS, METRICS (we skip RELATIONSHIPS and FACTS)
- Use matching field names (`s.revenue`, `s.cost`) for tutorial clarity per research recommendation

**Databricks tab (replace lines 48-56):** Replace with valid `CREATE VIEW ... WITH METRICS LANGUAGE YAML` syntax:

```sql
CREATE OR REPLACE VIEW sales
  WITH METRICS
  LANGUAGE YAML
  AS $$
    version: 1.1
    source: source_table
    dimensions:
      - name: country
        expr: country
      - name: region
        expr: region
    measures:
      - name: revenue
        expr: SUM(revenue)
      - name: cost
        expr: SUM(cost)
  $$;
```

Key syntax rules (from 18-RESEARCH.md):
- Command is `CREATE VIEW`, NOT `CREATE METRIC VIEW` -- `WITH METRICS` makes it a metric view
- `LANGUAGE YAML` is mandatory
- Body is YAML inside `$$ ... $$` delimiters
- Uses `dimensions` and `measures` arrays with `name` and `expr` keys
- No separate "facts" concept in Databricks YAML -- skip fact hint per research recommendation (deferred idea)
