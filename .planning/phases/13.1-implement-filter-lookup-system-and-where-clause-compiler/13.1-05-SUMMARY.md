## Decisions Made
- Used `Any` (not `ScalarT` TypeVar) for scalar lookup type args to satisfy basedpyright strict mode. A free TypeVar on class definitions caused `reportUnknownVariableType` and `reportInvalidTypeVarUse` errors in match/case destructuring and Field method annotations. Only tightened In (`Collection[Any]`) and Between (`tuple[Any, Any]`) where the type distinction is meaningful.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Reverted ScalarT TypeVar approach to maintain basedpyright strict compliance**
- **Found during:** Task 1 (type checking verification)
- **Issue:** Plan specified `ScalarT = TypeVar("ScalarT")` for scalar lookups, but basedpyright strict mode rejects free TypeVars in match/case destructuring (reportUnknownVariableType) and single-use TypeVars in non-generic methods (reportInvalidTypeVarUse). This produced 90 type errors.
- **Fix:** Kept scalar lookups as `Lookup[Any]` (matching prior working state), tightened only In to `Lookup[Collection[Any]]` and Between to `Lookup[tuple[Any, Any]]` where the type distinction is meaningful without introducing unresolved TypeVars.
- **Files modified:** src/cubano/filters.py, src/cubano/fields.py
- **Verification:** basedpyright 0 errors, 569 tests pass
- **Committed in:** 886536e (Task 1 commit)

**2. [Rule 1 - Bug] Fixed unused Not import in test_query.py after __ne__ change**
- **Found during:** Task 1 (lint check)
- **Issue:** Changing Field.__ne__ from Not(Exact()) to NotEqual made the Not import unused in test_query.py and the existing inequality test fail.
- **Fix:** Updated test assertion to check for NotEqual, removed unused Not import.
- **Files modified:** tests/unit/test_query.py
- **Verification:** ruff check passes, test passes
- **Committed in:** 886536e (Task 1 commit)
