  <behavior>
    Existing operator overloads (rewrite return types):
    - Field.__eq__(value) -> Exact(field_name=self.name, value=value)
    - Field.__ne__(value) -> Not(Exact(field_name=self.name, value=value))
    - Field.__lt__(value) -> Lt(field_name=self.name, value=value)
    - Field.__le__(value) -> Lte(field_name=self.name, value=value)
    - Field.__gt__(value) -> Gt(field_name=self.name, value=value)
    - Field.__ge__(value) -> Gte(field_name=self.name, value=value)

    New named methods:
    - Field.between(lo, hi) -> Between(field_name=self.name, value=(lo, hi))
    - Field.in_(values) -> In(field_name=self.name, value=values)
    - Field.like(pattern) -> Like(field_name=self.name, value=pattern)
    - Field.ilike(pattern) -> ILike(field_name=self.name, value=pattern)
    - Field.startswith(prefix) -> StartsWith(field_name=self.name, value=prefix)
    - Field.istartswith(prefix) -> IStartsWith(field_name=self.name, value=prefix)
    - Field.endswith(suffix) -> EndsWith(field_name=self.name, value=suffix)
    - Field.iendswith(suffix) -> IEndsWith(field_name=self.name, value=suffix)
    - Field.iexact(value) -> IExact(field_name=self.name, value=value)
    - Field.isnull() -> IsNull(field_name=self.name, value=True)

    Escape hatch:
    - Field.lookup(lookup_cls, value) -> lookup_cls(field_name=self.name, value=value)

    All methods:
    - Raise RuntimeError if self.name is None (before __set_name__)
    - Work on Metric, Dimension, and Fact field subclasses
    - Return Predicate subclasses (not Q objects)

    Composition still works via Predicate operators:
    - (Sales.country == "US") & (Sales.revenue > 1000) -> And(Exact(...), Gt(...))
    - (Sales.country == "US") | (Sales.country == "CA") -> Or(Exact(...), Exact(...))
    - ~(Sales.country == "US") -> Not(Exact(...))

    Field.__hash__ = object.__hash__ preserved (no change needed).
  </behavior>
