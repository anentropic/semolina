<objective>
Fix WHERE clause column name resolution so fields with explicit `source=` overrides emit the correct warehouse column name in the SQL WHERE clause, matching the behavior already implemented in SELECT and ORDER BY via `_resolve_col_name`.

Purpose: A field defined as `revenue_usd = Metric[int](source="revenue_usd")` currently emits `"REVENUE_USD"` (Snowflake-normalized Python name) in WHERE, while emitting `"revenue_usd"` (the source value) in SELECT — causing a SQL column name mismatch. This correctness gap was introduced in Phase 20.1 when `source=` was added to `Field` and `_resolve_col_name` was added for SELECT/ORDER BY without updating `_compile_predicate`.

Output: Three surgical changes (filters.py, fields.py, engines/sql.py) + regression tests in test_sql.py.
</objective>
