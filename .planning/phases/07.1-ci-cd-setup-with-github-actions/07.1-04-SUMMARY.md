## Implementation Details

### Task 1: Update Dependabot Config

**Changes:**
- Replaced `patterns: ["*"]` grouping with `dependency-type: "development"`
- Added `day: "monday"` to pip schedule for predictable update timing
- Renamed group from `python-dependencies` to `dev-dependencies` for clarity
- Kept github-actions ecosystem configuration unchanged

**Rationale:**
- Cubano has zero runtime dependencies (dependencies = [])
- Optional dependencies (databricks, snowflake) are user-installed, not tracked in CI
- Only dev dependencies need automated tracking
- Dependency-type grouping is more semantic than pattern matching
- Weekly Monday schedule reduces PR noise while maintaining security freshness

**Configuration:**
```yaml
- package-ecosystem: "pip"
  directory: "/"
  schedule:
    interval: "weekly"
    day: "monday"
  groups:
    dev-dependencies:
      dependency-type: "development"
```

## Key Decisions

1. **Use dependency-type: "development" over pattern matching**
   - More semantic and explicit about what's being tracked
   - Aligns with Dependabot's intended configuration model
   - Prevents accidental inclusion of production dependencies (if added later)

2. **Weekly Monday schedule for pip updates**
   - Balances security freshness with review burden
   - Predictable timing helps with sprint planning
   - Security updates still trigger immediately regardless of schedule

3. **Keep GitHub Actions tracking unchanged**
   - Pattern-based grouping works well for Actions (no dev vs prod distinction)
   - Existing configuration already functional
   - Unnecessary to change working configuration
