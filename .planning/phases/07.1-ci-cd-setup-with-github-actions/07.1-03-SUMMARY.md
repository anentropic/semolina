# Phase 07.1 Plan 03: Release Pipeline Summary

**One-liner:** Automated release pipeline with build validation and PyPI Trusted Publishing on semver tags

## What Was Built

Created a complete GitHub Actions release pipeline that automates the build, validation, and publishing process for Cubano packages to PyPI.

### Release Workflow (.github/workflows/release.yml)

**Trigger:** Semver tag pushes (v1.2.3, v1.2.3-rc1, etc.)

**Job Pipeline:**
1. **build** - Produces wheel and sdist distributions
   - Checks out repository with full git history
   - Syncs dependencies with uv
   - Builds distributions with `uv build`
   - Uploads artifacts for downstream jobs

2. **validate** - Tests distributions in isolated environments
   - Matrix: Python 3.11 and 3.14
   - Tests wheel installation and imports
   - Tests sdist installation and imports
   - Validates py.typed marker presence in wheel
   - Inspects wheel metadata for correctness

3. **changelog** - Generates release notes
   - Checks out full git history
   - Installs git-cliff
   - Generates CHANGELOG.md from conventional commits
   - Uploads changelog as artifact

4. **publish** - Publishes to PyPI
   - Depends on validate and changelog passing
   - Uses PyPI Trusted Publishing (OIDC)
   - No API tokens required
   - Publishes to https://pypi.org/p/cubano

### Changelog Configuration (.cliff.toml)

**Features:**
- Parses conventional commit messages
- Groups by type: Features, Bug Fixes, Documentation, Performance, Refactoring, Styling, Testing, Miscellaneous
- Filters out non-conventional commits
- Sorts commits oldest-first within groups
- Generates clean, categorized changelog entries

## Manual Setup Required

**IMPORTANT:** Before the first release, PyPI Trusted Publishing must be configured manually (one-time setup):

1. Sign in to PyPI (https://pypi.org)
2. Navigate to Account settings → Publishing
3. Add new publisher:
   - Repository owner: `anentropic`
   - Repository name: `cubano`
   - Workflow name: `release.yml`
   - Environment name: `pypi`
4. Save configuration

This enables password-less publishing via OIDC tokens generated by GitHub Actions.

## Quality Gates

**Hard Blocks:**
- Failed validation in any Python version blocks publish
- Missing py.typed marker blocks publish
- Failed wheel/sdist installation blocks publish

**No Override Path:**
- If validation fails, publish job will not run
- Manual intervention required to fix issues and re-tag

## Testing Recommendations

**Test Workflow:**
1. Create a test tag: `git tag v0.1.1-test && git push origin v0.1.1-test`
2. Monitor workflow in GitHub Actions
3. Verify job execution order
4. Check validation matrix runs for both Python versions
5. Confirm publish job waits for validation

**Test Validation Blocking:**
1. Temporarily remove py.typed marker from package
2. Push test tag
3. Verify validation fails and publish is skipped
4. Restore py.typed marker
