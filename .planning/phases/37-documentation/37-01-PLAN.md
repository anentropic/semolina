   ```
   Note: `fetch_arrow_table()` delegates to the underlying ADBC cursor. No extra dependencies beyond the backend driver.

3. **"Convert to a Pandas DataFrame"** section -- Show `table.to_pandas()`:

   df = table.to_pandas()

   ```
   Note: requires pandas installed (`pip install pandas`). PyArrow uses zero-copy conversion where column types allow it.

4. **"Convert to a Polars DataFrame"** section -- Show `pl.from_arrow(table)`:
   ```
   import polars as pl

   df = pl.from_arrow(table)
   print(type(df))
   # <class 'polars.dataframe.frame.DataFrame'>
   ```
   Note: requires polars installed (`pip install polars`). Zero-copy, no pandas dependency needed.

5. **"When to use Arrow output"** section -- Brief guidance:
   - Use `fetch_arrow_table()` when passing results to Pandas, Polars, or other Arrow-compatible tools
   - Use `fetchall_rows()` when working with individual rows or serializing to JSON

   | `read_only` | bool | No | Open in read-only mode (default: `false`) |

   Then show the standard load-and-register pattern:
   ```python
   from semolina import register, pool_from_config

   ```

   Add a `.. tip::` about named connections (same as snowflake.rst).

   Add a `.. note::` about pool_size: DuckDB defaults to `pool_size=1`. In-memory databases (`:memory:`) are isolated per connection, so `pool_size > 1` with `:memory:` raises a `ValidationError`. Use a file-backed database for multiple connections.

3. **"Configure manually"** -- Show manual construction:
   ```python
   from adbc_poolhouse import DuckDBConfig, create_pool
   from semolina import register

   config = DuckDBConfig(database="/path/to/warehouse.db")
   pool = create_pool(config)
   register("default", pool, dialect="duckdb")
   ```

4. **"Run a query"** -- Same as snowflake.rst pattern:
   ```python
   from semolina import SemanticView, Metric, Dimension


   class Sales(SemanticView, view="sales"):
       revenue = Metric()
       country = Dimension()


   cursor = (
       Sales.query()
       .metrics(Sales.revenue)
       .dimensions(Sales.country)
       .execute()
   )
   for row in cursor.fetchall_rows():
       print(row.country, row.revenue)
   ```

5. **"Generated SQL"** -- Explain that DuckDB uses `semantic_view()` table function (different from Snowflake AGG() / Databricks MEASURE()):
   ```sql
   SELECT *

   ```

   Add a second example for row-level (facts) queries:
   ```sql
   SELECT *
   FROM semantic_view('sales', facts := ['unit_price', 'quantity'])
   ```
