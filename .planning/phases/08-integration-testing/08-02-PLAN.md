<task type="auto">
  <name>Task 2: Add warehouse connection fixtures with per-worker isolation</name>
  <files>tests/conftest.py</files>
  <action>
Add warehouse connection fixtures to tests/conftest.py after existing credential fixtures:

1. **test_schema_name fixture (scope="session"):**
   - Accepts worker_id parameter (provided by pytest-xdist)
   - Returns "cubano_test_main" if worker_id == "master" (non-parallel execution)
   - Returns f"cubano_test_{worker_id}" for parallel workers (e.g., "cubano_test_gw0", "cubano_test_gw1")
   - Docstring: "Generate unique schema name for this worker to prevent parallel test conflicts."

2. **snowflake_connection fixture (scope="session"):**
   - Depends on: snowflake_credentials, test_schema_name
   - Lazy import: `import snowflake.connector` inside fixture (not at module level)
   - Create connection using credentials: snowflake.connector.connect(account=..., user=..., password=credentials.password.get_secret_value(), warehouse=..., database=..., role=...)
   - Setup: cursor.execute(f"CREATE SCHEMA IF NOT EXISTS {test_schema_name}"), cursor.execute(f"USE SCHEMA {test_schema_name}")
   - Yield connection
   - Teardown (in finally block): cursor.execute(f"DROP SCHEMA IF EXISTS {test_schema_name} CASCADE"), cursor.close(), conn.close()
   - Wrap teardown in try/except to print warning if cleanup fails (don't raise to avoid masking test failures)
   - Docstring: "Create Snowflake connection with isolated test schema. Schema isolation prevents parallel test conflicts. Session scope reduces connection overhead."

3. **databricks_connection fixture (scope="session"):**
   - Same pattern for Databricks using databricks.sql.connect
   - Schema naming: f"{catalog}.{test_schema_name}" for Unity Catalog
   - Docstring: "Create Databricks connection with isolated test schema in Unity Catalog."

4. **mock_engine fixture (scope="function"):**
   - Returns MockEngine() instance
   - Function scope ensures each test gets fresh engine (no state leakage)
   - Docstring: "Function-scoped MockEngine for isolated fast tests without warehouse connection."

Keep existing fixtures unchanged (clean_registry, sales_model, sales_fixtures, sales_engine).

Implementation notes:
- Use `worker_id` fixture from pytest-xdist (automatically available in parallel mode)
- Session scope for warehouse connections reduces expensive connection overhead
- Teardown with yield pattern guarantees cleanup even on test failures
- finally block with try/except in teardown ensures cleanup attempts don't mask test failures
