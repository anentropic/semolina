    Read src/cubano/engines/sql.py in full before editing.

    **1. Add abstract normalize_identifier to Dialect ABC:**
    ```python
    @abstractmethod
    def normalize_identifier(self, name: str) -> str:
        """
        Fold a Python field name to the SQL column name the warehouse stores.

        Called when field.source is None. Each dialect knows its default identifier
        folding: Snowflake stores unquoted identifiers as UPPERCASE; Databricks as
        lowercase; Mock passes through unchanged.
        """
    ```

    **2. Implement on each concrete Dialect subclass:**
    - SnowflakeDialect: `return name.upper()`
    - DatabricksDialect: `return name.lower()`
    - MockDialect: `return name` (identity)

    **3. Add _resolve_col_name helper to SQLBuilder:**
    ```python
    def _resolve_col_name(self, field: Any) -> str:
        """Resolve SQL-facing column name: source takes priority, else normalize."""
        assert field.name is not None
        if field.source is not None:
            return field.source
        return self.dialect.normalize_identifier(field.name)
    ```

    **4. Update _build_select_clause:** Replace every direct use of `field.name` (or the
    equivalent string that feeds into wrap_metric/quote_identifier) with
    `self._resolve_col_name(field)`. Read the existing method carefully — it may use
    `field.name` in multiple places (metrics, dimensions, facts).

    **5. Update _build_order_by_clause:** Same pattern — use `_resolve_col_name` for the
    ORDER BY column reference.

    **6. Update _compile_predicate (WHERE clause):** Predicates store the Python field_name
    string (not the Field descriptor). Apply normalize_identifier to it before quoting:
    ```python
    case Exact(field_name=f, value=v):
        normalized = self.dialect.normalize_identifier(f)
        return f"{q(normalized)} = {ph}", [v]
    ```
    Apply this pattern to ALL Lookup branch cases in _compile_predicate — not just Exact.
    Every branch that calls q(f) where f is a field_name from a Lookup must normalize first.

    **7. Update tests/unit/test_sql.py:**
    - Add tests for normalize_identifier on each dialect (3 cases each)
    - Add test for _resolve_col_name: field with source= uses source verbatim, field without
      source uses normalize_identifier
    - Update existing tests that assert SQL output: after normalization, Snowflake SELECT
      should use "ORDER_ID" (uppercase) for a field named 'order_id'. Check whether existing
      test models use MockDialect — if so, normalize_identifier is identity and no change needed.
    - Add test: WHERE clause with Snowflake dialect normalizes field_name to uppercase

    Do not change the behavior of existing tests using MockDialect (identity transform means
    test assertions stay the same).
